
# 圆运阁页顶部导航_设计说明.md

**文档创建日期：2026年01月27日**
**设计状态：正式设计规范**

## 设计需求

### 设计主题
专业、现代且交互丰富的顶部导航栏设计，体现圆运阁作为专业易经占卜文化调性，强调清晰、直观与动态效果，符合现代Web设计趋势。

### 适配设备
**PC网页端**，标准宽度：1440px（桌面端主流分辨率，确保最佳展示效果）
**响应式适配**：设计需适配从大屏桌面到1280宽度的完整断点范围。

### 页面布局结构
本设计说明聚焦于页面顶部的全局导航栏组件，其结构如下：

- **总容器（导航栏区域）**：
  - 高度：固定72px（提供足够的视觉权重和点击区域）
  - 布局：水平Flex布局，内容左右分布，垂直居中
  - 定位：默认采用固定定位（`position: fixed; top: 0;`），随页面滚动始终悬浮在顶部，背景色可随滚动渐变。

- **左侧区域（品牌与主导航）**：
  1. **圆运阁Logo**：放置于最左侧，链接至首页，尺寸约40px×40px，清晰可辨。
  2. **主导航菜单**：Logo右侧水平排列，间距均匀。包含以下典型菜单项（示例）：
     - 广场
     - 排盘（可设计为带下拉箭头的二级菜单）
     - 藏经阁
     - TEST

- **右侧区域（用户操作与CTA）**：
  1. **辅助功能入口**：如“帮助中心”、“中文/EN”语言切换。
  2. **行动号召按钮（CTA）**：
     - “登录”按钮（次要按钮样式）
     - “免费试用”按钮（主按钮样式，视觉突出）

### 颜色方案
- **主色调（品牌色）**：采用**国风**作为主色，色值建议为 `#1890FF` 或 `#007AFF`。用于Logo、主按钮背景、高亮菜单项。
- **导航栏背景色**：
  - **默认状态**：纯白色 `#FFFFFF`，或极浅的灰色 `#F9FAFB`，营造干净背景。
  - **滚动状态**：当页面滚动超过一定阈值（如50px），背景色可轻微加深或增加半透明毛玻璃效果（`backdrop-filter: blur(10px); background-color: rgba(255, 255, 255, 0.95);`），以增强层级感。
- **文字颜色**：
  - **菜单项默认**：深灰色 `#333333` 或 `#1F2937`。
  - **菜单项悬停/激活**：主色调（科技蓝）。
  - **菜单项禁用**：浅灰色 `#9CA3AF`。
- **辅助色与状态色**：
  - **次要按钮边框/文字**：中灰色 `#D1D5DB`。
  - **成功/确认**：绿色 `#10B981`。
  - **错误/警告**：红色 `#EF4444`。
  - **分割线**：浅灰色 `#E5E7EB`。

### 排版和字体
- **字体家族**：采用现代、无衬线字体，优先使用系统字体栈以确保性能和一致性，例如：`-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif`。
- **字号与字重**：
  - **导航菜单项**：常规为 `16px`，字重 `400`（常规）或 `500`（中等）。
  - **激活/当前页菜单项**：字号保持 `16px`，字重可增加至 `600`（半粗），并使用主色调。
  - **按钮文字**：`14px` 或 `16px`，字重 `500` 或 `600`。
  - **Logo文字（如有）**：根据Logo设计，通常更大或更具特色。
- **行高与间距**：
  - 菜单项行高：`1.5`，确保垂直居中。
  - 菜单项水平间距：`32px` 或 `40px`，提供舒适的点击区域和视觉呼吸感。

### 组件和元素
- **导航菜单项（链接）**：
  - 样式：内联块级元素，无下划线。
  - 悬停效果：文字颜色变为主色，可伴随轻微的下划线或底部边框颜色变化。
  - 激活状态：更深的颜色或底部实线边框（`border-bottom: 2px solid #1890FF;`）。
- **主按钮（如“免费试用”）**：
  - 样式：主色背景，白色文字。
  - 尺寸：高度约 `40px`，水平内边距约 `20px`。
  - 圆角：`6px`（中等圆角，现代感）。
  - 阴影：轻微投影 `box-shadow: 0 2px 4px rgba(0, 122, 255, 0.2)`。
- **次要按钮（如“登录”）**：
  - 样式：白色背景，主色文字和边框。
  - 尺寸：与主按钮保持一致。
  - 圆角：`6px`。
- **下拉菜单（二级导航）**：
  - 触发方式：鼠标悬停或点击（根据交互设计）。
  - 容器：白色背景，`8px` 圆角，中等投影 `box-shadow: 0 10px 15px -3px rgba(0, 0, 0, 0.1)`。
  - 菜单项：垂直排列，内边距充足。
- **汉堡菜单图标（移动端）**：
  - 尺寸：`24px × 24px`。
  - 线条：三条横线，颜色为深灰色。
  - 交互：点击后三条线可动画变化为“X”或箭头。

### 交互效果
**核心：充分利用墨刀平台倡导的动态组件与状态管理理念。**

1.  **菜单项状态切换**：
    - **默认** -> **悬停**：文字颜色变化，可伴随下划线宽度从0到100%的CSS过渡动画（`transition: color 0.2s ease, border-bottom 0.2s ease;`）。
    - **点击/激活**：状态持久化，样式区别于悬停（如更粗的底部边框）。

2.  **导航栏背景滚动渐变**：
    - 监听页面滚动，当 `scrollTop > 50px` 时，通过添加/移除CSS类或直接修改样式，将导航栏背景从透明/纯白渐变为半透明毛玻璃效果。使用 `transition` 实现平滑过渡。

3.  **下拉菜单展示**：
    - 悬停触发：延迟约 `200ms` 显示，防止误触；使用 `opacity` 和 `transform: translateY(-5px)` 实现淡入和轻微上移动画。
    - 点击触发（更适合移动端）：点击后菜单向下展开，带动画。

4.  **移动端侧滑导航**：
    - 点击汉堡图标，从右侧或左侧滑入一个全屏或大半屏的导航抽屉。
    - 抽屉内部菜单垂直排列，包含所有主导航项和用户操作按钮。
    - 支持手势滑动关闭。

### 边界处理
- **PC网页（单页应用或常规页面）**：导航栏作为页面的一部分，通常不需要额外的外部边框。其与下方内容的边界通过背景色变化、阴影或一条细分割线（`border-bottom: 1px solid #E5E7EB;`）来清晰界定。
- **背景色区分**：导航栏背景色（白色/浅色）与页面主体内容背景色应有轻微区分或通过阴影建立层级，确保导航栏的悬浮感和独立性。

### 响应式设计规范
- **断点定义**：
  - `≥ 1024px`：桌面端，显示完整导航栏。
  - `768px - 1023px`：平板端，可能略微缩小菜单间距，或考虑折叠部分次要菜单项。
  - `< 768px`：移动端，隐藏水平导航菜单和右侧部分按钮，显示汉堡菜单图标。
- **实现方式**：使用CSS媒体查询（`@media`）进行布局切换。
- **触摸优化**：在移动端，所有点击目标（菜单项、按钮）的高度至少为 `44px`，符合人机交互指南。
- **视口设置**：确保HTML中包含 `<meta name="viewport" content="width=device-width, initial-scale=1">`。

### 开发与复用建议
- **组件化**：建议将整个导航栏封装为独立的Vue/React/Web Component，便于在整个网站或应用内复用。
- **状态管理**：当前激活的页面（菜单项）状态应由路由或应用状态管理，确保导航状态准确。
- **性能**：滚动监听事件应进行防抖处理，背景变化动画应使用CSS `transform` 和 `opacity` 属性以利用GPU加速。
- **可访问性**：
  - 为导航菜单添加正确的ARIA标签（如 `aria-label="主导航"`）。
  - 确保可以通过键盘Tab键遍历所有导航项。
  - 下拉菜单在获得焦点时也应能正确展开。
