# 六爻排盘项目开发推进步骤说明书

# 一、项目现状分析

## 1.1 已实现功能
- **前端**：React 19.2.3 基础架构，六爻排盘核心组件（多种起卦方法）
- **后端**：FastAPI 0.104.1 基础架构，六爻排盘算法实现
- **依赖**：前端 lunar-javascript，后端 lunar-python 历法计算库

## 1.2 缺失功能
- **用户体系**：注册、登录、验证码发送
- **排盘记录**：保存、查询、分页
- **评论功能**：添加、查询、修改、删除
- **数据库**：SQLite 集成与表结构设计
- **安全机制**：JWT Token、密码哈希

# 二、开发推进步骤

## 2.1 后端开发步骤

### 阶段一：数据库集成与模型设计
1. **集成 SQLite 数据库**
   - 安装 SQLAlchemy 依赖
   - 配置数据库连接
   - 创建数据库初始化脚本

2. **创建核心数据模型**
   - `user.py`：用户表模型（id, phone, password, ext_info, create_time, update_time, is_active）
   - `pan_record.py`：排盘记录表模型（id, user_id, pan_type, pan_params, pan_result, create_time, supplement）
   - `comment.py`：评论表模型（id, pan_record_id, user_id, content, create_time, update_time, is_public, ext_info）

3. **实现模型关联与索引**
   - 用户表：phone 字段唯一索引
   - 排盘记录表：user_id 与 pan_type 联合索引
   - 评论表：pan_record_id、user_id 联合索引，create_time 索引

### 阶段二：核心工具类实现
1. **实现密码哈希工具**
   - 使用 bcrypt 库
   - 实现密码加密与验证方法

2. **实现 JWT Token 工具**
   - 生成 Token（有效期7天）
   - 验证 Token 有效性
   - 解析 Token 中的用户信息

3. **实现验证码工具**
   - 生成6位数字验证码
   - 验证码存储与过期管理（5分钟）
   - 验证码发送模拟（控制台输出）

### 阶段三：用户相关接口实现
1. **实现验证码发送接口**
   - `/api/v1/user/send_code` POST
   - 参数：phone
   - 返回：发送状态

2. **实现用户注册接口**
   - `/api/v1/user/register` POST
   - 参数：phone, code, password
   - 返回：用户ID

3. **实现用户登录接口**
   - `/api/v1/user/login` POST
   - 支持两种方式：phone+code 或 phone+password
   - 返回：token, user_id, phone

### 阶段四：排盘记录接口实现
1. **实现排盘记录保存接口**
   - `/api/v1/pan/save` POST（需登录）
   - 参数：pan_type, pan_params, pan_result, supplement
   - 返回：record_id

2. **实现排盘记录查询接口**
   - `/api/v1/pan/list` GET（需登录）
   - 参数：pan_type, page, size
   - 返回：排盘记录列表（带分页）

### 阶段五：评论接口实现
1. **实现评论添加接口**
   - `/api/v1/comment/add` POST（需登录）
   - 参数：pan_record_id, content, is_public
   - 返回：comment_id

2. **实现评论查询接口**
   - `/api/v1/comment/list` GET（需登录）
   - 参数：pan_record_id, page, size
   - 返回：评论列表

3. **实现评论更新接口**
   - `/api/v1/comment/update` POST（需登录）
   - 参数：comment_id, content, is_public
   - 返回：更新状态

4. **实现评论删除接口**
   - `/api/v1/comment/delete` POST（需登录）
   - 参数：comment_id
   - 返回：删除状态

### 阶段六：中间件与安全实现
1. **实现 Token 验证中间件**
   - 保护需要登录的接口
   - Token 过期处理

2. **实现请求频率限制**
   - 验证码发送：60秒限制
   - 登录/注册：防止暴力破解

3. **实现参数验证**
   - 使用 Pydantic 进行请求参数验证
   - 统一错误返回格式

# 三、技术实现细节

## 3.1 后端技术实现
- **框架**：FastAPI 0.104.1（替代 Flask，性能更优）
- **数据库**：SQLAlchemy + SQLite
- **认证**：PyJWT + bcrypt
- **验证**：Pydantic V2

## 3.2 前端技术实现
- **框架**：React 19.2.3
- **路由**：React Router 6.x
- **请求**：Axios
- **状态**：React Context API
- **样式**：CSS Modules

# 四、开发规范

## 4.1 代码规范
- **文件注释**：每个文件顶部添加标准注释
- **命名规范**：驼峰命名法
- **代码风格**：遵循 ESLint/Flake8 规范

## 4.2 接口规范
- **前缀**：统一使用 `/api/v1/`
- **返回格式**：
  ```json
  {
    "code": 200,
    "msg": "success",
    "data": {}
  }
  ```
- **状态码**：200成功，400参数错误，401未登录，500服务器错误

## 4.3 安全规范
- **密码**：bcrypt 哈希存储
- **Token**：JWT 加密，7天有效期
- **参数**：严格验证，防止注入攻击
- **频率**：限制敏感接口请求频率

# 七、风险评估与应对策略

## 7.1 潜在风险
- **数据库迁移**：后期可能需要迁移到 MySQL
- **性能优化**：用户量增长后的响应速度
- **安全漏洞**：Token 管理与密码存储

## 7.2 应对策略
- **数据库抽象**：使用 ORM 框架，便于后期迁移
- **缓存机制**：关键数据缓存，提升响应速度
- **安全审计**：定期检查安全漏洞，更新依赖库
