# PanRecord 排盘记录表字段分析文档

## 一、文档概述

本文档详细描述六爻排盘系统中 `pan_record`（排盘记录）数据表的完整字段结构及业务含义。该表用于存储用户的排盘记录信息，包括排盘参数、排盘结果、审核状态等核心数据。

### 1.1 基本信息

| 项目 | 说明 |
|------|------|
| **数据表名称** | pan_record |
| **所属模块** | 后端数据模型 |
| **模型文件路径** | backend/app/models/pan_record.py |
| **数据库类型** | SQLite |
| **数据库文件** | backend/app/db/yyggywh.db |
| **主键** | id (自增整数) |

### 1.2 表结构概览

| 序号 | 字段名称 | 数据类型 | 约束类型 | 可为空 | 默认值 |
|------|----------|----------|----------|--------|--------|

## 二、字段详细说明

### 2.1 主键字段

#### 字段 1：id

| 属性 | 值 |
|------|-----|
| **字段名称** | id |
| **数据 SQLAlchemy 类型** | Integer |
| **数据库实际类型** | INTEGER |
| **长度** | - |
| **约束条件** | PRIMARY KEY, AUTOINCREMENT, INDEX |
| **默认值** | 自增（自动分配） |
| **是否可为空** | 否 |
| **业务含义** | 排盘记录的唯一标识符，作为表的主键，用于定位和关联单条排盘记录 |

**详细说明**：
- 该字段为自增整数类型，每次插入新记录时自动分配
- 数值从 1 开始，依次递增
- 作为主键，确保每条记录的唯一性
- 创建了索引（index=True），提升按 ID 查询的性能
- 在 API 响应中返回该字段，供前端进行记录关联和操作

### 2.2 用户关联字段

#### 字段 2：user_id

| 属性 | 值 |
|------|-----|
| **字段名称** | user_id |
| **数据 SQLAlchemy 类型** | Integer |
| **数据库实际类型** | INTEGER |
| **长度** | - |

- 必须指定用户 ID，不允许为空
- 每个排盘记录必须归属于某个注册用户
- 创建了索引（联合索引 idx_pan_user_type），提升按用户查询的性能
- 通过该字段可查询某个用户的所有排盘记录
- 在用户中心页面展示该用户的排盘历史时使用此字段进行关联查询

**外键关系**：
```python
user_id = Column(Integer, ForeignKey("user.id"), nullable=False)
```

### 2.5 时间管理字段

#### 字段 6：create_time

| 属性 | 值 |
|------|-----|
| **字段名称** | create_time |
| **数据 SQLAlchemy 类型** | Integer |

| **是否可为空** | 是 |
| **业务含义** | 记录排盘记录的创建时间，使用 Unix 时间戳格式 |

**详细说明**：
- 存储格式为 Unix 时间戳（秒级），例如：1741161600
- 创建记录时自动填充当前时间戳
- 可转换为人类可读的日期时间格式进行展示
- 用于按时间排序、筛选历史记录
- 支持按时间范围查询排盘记录

**时间戳转换示例**：
```python
import datetime
create_timestamp = 1741161600
readable_time = datetime.datetime.fromtimestamp(create_timestamp)
# 结果：2026-03-05 00:00:00
```

#### 字段 14：deleted_at

| 属性 | 值 |
|------|-----|
| **字段名称** | deleted_at |
| **数据 SQLAlchemy 类型** | Integer |
| **数据库实际类型** | INTEGER |
| **长度** | - |
| **约束条件** | NULLABLE |
| **默认值** | NULL |
| **是否可为空** | 是 |
| **业务含义** | 软删除标记字段，存储记录被删除的时间戳 |

**详细说明**：
- 采用软删除机制，保护数据安全
- 当记录被删除时，填充当前时间戳作为删除标记
- 为 NULL 时表示记录未被删除

**软删除机制**：
```python
# 逻辑删除
if record.deleted_at is None:
    # 记录未被删除，正常显示
else:
    # 记录已删除，不显示（可通过配置实现）
```

**与 is_visible 的关系**：
- `deleted_at` 表示物理层面的删除状态
- `is_visible` 表示业务层面的显示状态
- 两者可以独立使用，也可以配合使用实现更复杂的可见性控制

**恢复机制**：
- 由于采用软删除，被误删的记录可以恢复
- 只需将 `deleted_at` 设为 NULL 即可恢复记录

### 4.2 SQLAlchemy 关联定义

```python
# 关联创建记录的用户
user = relationship("User", foreign_keys=[user_id], back_populates="pan_records")

# 关联执行审核的用户
audit_user = relationship("User", foreign_keys=[audit_user_id])

# 关联评论（级联删除）
comments = relationship("Comment", back_populates="pan_record", cascade="all, delete-orphan")
```

- **级联删除**：当排盘记录被删除时，关联的评论记录也会被自动删除（`cascade="all, delete-orphan"`）

### 7.1 SQL 建表语句（参考）

```sql
CREATE TABLE IF NOT EXISTS pan_record (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    pan_type VARCHAR(20) DEFAULT 'liuyao',
    pan_params TEXT NOT NULL,
    pan_result TEXT NOT NULL,
    create_time INTEGER DEFAULT (strftime('%s', 'now')),

    audit_remark TEXT,

    FOREIGN KEY (user_id) REFERENCES user(id),
    FOREIGN KEY (audit_user_id) REFERENCES user(id)
);

CREATE INDEX idx_pan_user_type ON pan_record(user_id, pan_type);
```
