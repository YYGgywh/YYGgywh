# CSS Modules 样式改造项目计划书

## 1. 项目背景

### 1.1 现状分析

当前项目存在严重的CSS污染问题，主要表现为：

- **类名冲突**：多个组件使用相同的类名（如 `.card`、`.button`、`.container` 等），导致样式相互覆盖
- **样式耦合**：组件样式之间存在复杂的依赖关系，修改一个组件的样式可能影响其他组件
- **维护困难**：全局样式难以追踪和管理，调试和定位问题耗时较长
- **团队协作效率低**：不同开发者的样式代码容易产生冲突，代码合并困难

### 1.2 技术现状

- **项目架构**：React + React Router + Axios
- **构建工具**：Create React App (基于Webpack)
- **CSS使用情况**：
  - 总计56个CSS文件（排除node_modules和build目录）
  - 传统CSS文件：52个（92.9%）
  - CSS Modules文件：4个（7.1%）

### 1.3 改造必要性

1. **解决样式冲突**：通过CSS Modules的作用域隔离特性，彻底解决类名冲突问题
2. **提高可维护性**：样式与组件紧密关联，便于定位和维护
3. **改善开发体验**：减少样式调试时间，提高开发效率
4. **适应现代前端开发**：符合当前前端开发的最佳实践

## 8. 附录

### 8.1 CSS Modules开发规范

#### 8.1.1 文件命名规则
- **组件样式文件**：与组件同名，后缀为 `.module.css`，例：`AAA.jsx` 对应 `AAA.module.css`
- **全局样式文件**：使用普通 CSS 后缀，例：`global.css`（重置样式）、`common.css`（通用类名）
- **第三方样式覆盖文件**：命名为 `custom-[插件名].css`，例：`custom-antd.css`

#### 8.1.2 类名使用规则
- **组件私有类名**：在 `.module.css` 中定义，JSX 中通过 `styles.类名` 引用
- **类名命名**：采用短横线或小驼峰，仅包含字母/数字，例：`btn-primary`、`cardTitle`
- **类名透传**：可复用组件需预留 `className` 属性，合并私有类名与父组件传参
- **全局类名**：仅在普通 CSS 文件中定义，JSX 中直接以字符串引用

#### 8.1.3 样式差异化适配规则
- **默认值定义**：组件内可差异化的样式，通过CSS变量在组件根类名中设置默认值
- **父组件覆盖**：父组件通过传参类名，重定义CSS变量
- **特殊覆盖**：仅在未定义CSS变量时，使用 `:global()` 定位子组件原始类名

#### 8.2.2 JSX文件改造示例

**改造前（LikeButton.jsx）：**
```javascript
import React from 'react';
import './LikeButton.css';

const LikeButton = ({ isLiked, className }) => (
  <div className={`like-button ${isLiked ? 'liked' : ''} ${className || ''}`}>
    {/* 内容 */}
  </div>
);
```

**改造后（LikeButton.jsx）：**
```javascript
import React from 'react';
import styles from './LikeButton.module.css';

const LikeButton = ({ isLiked, className }) => (
  <div className={`${styles.likeButton} ${isLiked ? styles.liked : ''} ${className || ''}`}>
    {/* 内容 */}
  </div>
);
```

### 8.3 常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| **动态类名如何处理** | 使用模板字符串：`${styles[className]}` |
| **第三方组件样式如何覆盖** | 使用 `:global()` 选择器：`.root :global(.ant-btn) { }` |
| **全局样式如何管理** | 保持Reset.css、variables.css等为全局CSS |
| **类名透传如何实现** | 合并类名：`${styles.root} ${className || ''}` |
| **调试时如何识别类名** | 使用React DevTools查看组件样式 |
