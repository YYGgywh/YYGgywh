## 数据类型说明

### 1. Integer 类型

- **用途**：存储整数类型数据，如ID、状态、计数等
- **范围**：根据数据库实现而定，通常为-2^31到2^31-1
- **示例**：id, gender, role, status

### 2. String 类型

- **用途**：存储字符串类型数据，如手机号、邮箱、登录名等
- **长度**：根据字段含义设置合适的长度限制
- **示例**：phone(11), login_name(20), email(50), avatar(255)

### 3. Boolean 类型

- **用途**：存储布尔值，如是否活跃、是否老用户等
- **取值**：True/False
- **示例**：is_active, is_old_user

### 4. 时间戳字段

- **用途**：存储时间信息
- **格式**：Unix时间戳（秒）
- **示例**：create_time, update_time, last_login_time, deleted_at

## 使用规范

### 1. 登录凭证

- 用户可以通过手机号、登录名或邮箱进行登录
- 手机号为必填字段，登录名和邮箱为选填
- 微信OpenID用于微信登录

### 2. 个人信息

- 出生日期相关字段用于排盘计算
- 支持公历和农历两种日历类型
- 昵称用于系统内显示，支持修改但有频率限制

### 3. 密码管理

- 密码使用哈希存储，不可明文存储
- 管理员和超级管理员有独立的后台登录密码

### 4. 权限管理

- 基于role字段实现权限控制：
  - 0=普通用户：只能操作自己的数据
  - 1=管理员：可以管理普通用户和内容
  - 99=超级管理员：拥有所有权限

### 5. 状态管理

- status字段控制账号是否可用（0=禁用, 1=正常）
- is_active字段标记账号是否活跃
- deleted_at字段实现软删除机制

### 6. 操作限制

- 登录名修改：一年最多4次
- 昵称修改：一年最多12次
- 系统通过modify_count和modify_time字段进行限制

## 注意事项

1. **数据安全**：密码等敏感信息必须使用哈希存储，禁止明文存储
2. **性能优化**：对频繁查询的字段（如phone、login_name、email）已创建索引
3. **数据一致性**：出生日期相关字段应保持一致性，确保日历类型与具体日期匹配
4. **操作限制**：严格执行登录名和昵称的修改频率限制
5. **软删除**：使用deleted_at字段实现软删除，查询时需过滤已删除记录
6. **权限控制**：基于role字段实现严格的权限控制，防止越权操作
