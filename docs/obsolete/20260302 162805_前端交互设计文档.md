# 用户注册/登录功能扩展与优化 前端交互设计文档

## 一、现有前端功能分析

### 1.1 现有登录/注册页面

- **页面结构**：包含登录和注册两个标签页
- **登录方式**：支持手机号+验证码或手机号+密码登录
- **注册方式**：仅支持手机号+验证码注册
- **交互流程**：
  1. 用户输入手机号
  2. 点击发送验证码
  3. 输入验证码和密码
  4. 点击登录/注册按钮

### 1.2 现有用户中心页面

- **页面结构**：显示用户基本信息
- **功能**：基本的用户信息展示

## 二、前端交互设计方案

### 2.1 登录/注册页面优化

#### 2.1.1 页面结构

- **顶部标签**：登录、注册
- **登录方式标签**：登录名登录、邮箱登录、手机登录、微信登录
- **注册方式标签**：手机注册、邮箱注册、微信注册

#### 2.1.2 登录流程

1. **登录名登录**：
   - 输入登录名和密码
   - 点击登录按钮
   - 验证成功后跳转到用户中心

2. **邮箱登录**：
   - 输入邮箱和密码
   - 点击登录按钮
   - 验证成功后跳转到用户中心

3. **手机登录**：
   - 输入手机号
   - 选择验证码或密码登录
   - 输入验证码或密码
   - 点击登录按钮
   - 验证成功后跳转到用户中心

4. **微信登录**：
   - 点击微信登录按钮
   - 弹出微信扫码弹窗
   - 用户扫码授权
   - 后端验证OpenID
   - 若已绑定账号，直接登录
   - 若未绑定账号，跳转到信息完善页面

#### 2.1.3 注册流程

1. **手机注册**：
   - 输入手机号
   - 点击发送验证码
   - 输入验证码
   - 输入密码和确认密码
   - 输入登录名（可选，不填则自动生成）
   - 点击注册按钮
   - 注册成功后跳转到用户中心

2. **邮箱注册**：
   - 输入邮箱
   - 点击发送验证码
   - 输入验证码
   - 输入密码和确认密码
   - 输入登录名（可选，不填则自动生成）
   - 点击注册按钮
   - 注册成功后跳转到用户中心

3. **微信注册**：
   - 点击微信注册按钮
   - 弹出微信扫码弹窗
   - 用户扫码授权
   - 跳转到信息完善页面
   - 输入登录名（可选，不填则自动生成）
   - 输入手机号或邮箱（二选一）
   - 点击确认按钮
   - 注册成功后跳转到用户中心

### 2.2 信息完善页面

#### 2.2.1 页面结构

- **标题**：完善用户信息
- **登录名输入框**（可选）
- **手机输入框**（可选，与邮箱二选一）
- **邮箱输入框**（可选，与手机二选一）
- **确认按钮**

#### 2.2.2 交互流程

1. 用户输入登录名（可选）
2. 用户输入手机号或邮箱（二选一）
3. 点击确认按钮
4. 后端验证信息
5. 信息验证成功后完成注册并登录
6. 跳转到用户中心

### 2.3 老用户完善登录名弹窗

#### 2.3.1 弹窗结构

- **标题**：完善登录名
- **提示信息**：请设置登录名，用于后续多渠道登录
- **登录名输入框**
- **确认按钮**
- **取消按钮**

#### 2.3.2 交互流程

1. 老用户首次登录后弹出
2. 用户输入登录名
3. 点击确认按钮
4. 后端验证登录名唯一性
5. 验证成功后更新用户信息
6. 弹窗关闭，跳转到用户中心

### 2.4 用户中心页面优化

#### 2.4.1 页面结构

- **用户基本信息**：头像、登录名、昵称、手机、邮箱
- **功能模块**：
  - 头像上传
  - 昵称修改
  - 微信绑定/解绑
  - 手机绑定/修改
  - 邮箱绑定/修改

#### 2.4.2 交互流程

1. **头像上传**：
   - 点击头像区域
   - 弹出文件选择器
   - 选择图片文件
   - 显示上传进度
   - 上传成功后显示新头像
   - 后端保存头像URL
   - 页面刷新

2. **昵称修改**：
   - 点击昵称修改按钮
   - 弹出修改昵称弹窗
   - 输入新昵称
   - 点击确认按钮
   - 后端验证并更新
   - 弹窗关闭，页面刷新

3. **微信解绑**：
   - 点击微信解绑按钮
   - 弹出验证弹窗
   - 输入密码或验证码
   - 点击确认按钮
   - 后端验证并解绑
   - 弹窗关闭，页面刷新

3. **手机/邮箱绑定**：
   - 点击绑定按钮
   - 弹出绑定弹窗
   - 输入手机号/邮箱
   - 点击发送验证码
   - 输入验证码
   - 点击确认按钮
   - 后端验证并绑定
   - 弹窗关闭，页面刷新

## 三、前端技术实现

### 3.1 技术栈

- **框架**：React 19.2.3
- **路由**：React Router 7.13.1
- **HTTP客户端**：axios 1.13.5
- **表单处理**：react-hook-form 7.71.2
- **状态管理**：React Context

### 3.2 组件设计

| 组件名称 | 功能 | 实现方式 |
|---------|------|----------|
| LoginPage | 登录/注册页面 | 包含多个子组件，支持多方式切换 |
| LoginForm | 登录表单 | 根据登录方式显示不同输入框 |
| RegisterForm | 注册表单 | 根据注册方式显示不同输入框 |
| WechatLogin | 微信登录组件 | 生成微信扫码二维码 |
| InfoComplete | 信息完善页面 | 用于微信注册和老用户完善信息 |
| UserCenter | 用户中心页面 | 显示和管理用户信息 |
| AvatarUpload | 头像上传组件 | 上传和预览用户头像 |
| NicknameEdit | 昵称修改组件 | 弹窗形式，修改用户昵称 |
| WechatUnbind | 微信解绑组件 | 弹窗形式，解除微信绑定 |
| PhoneBind | 手机绑定组件 | 弹窗形式，绑定/修改手机号 |
| EmailBind | 邮箱绑定组件 | 弹窗形式，绑定/修改邮箱 |

### 3.3 API调用

- **验证码发送**：调用 `/api/v1/user/send_code` 和 `/api/v2/verify-code/email`
- **注册**：调用 `/api/v1/user/register`、`/api/v2/register/email` 和 `/api/v2/register/wechat`
- **登录**：调用 `/api/v1/user/login`、`/api/v2/login/login-name`、`/api/v2/login/email`、`/api/v2/login/phone` 和 `/api/v2/login/wechat`
- **用户信息**：调用 `/api/v2/user/perfect-login-name`、`/api/v2/user/update-nickname`、`/api/v2/user/unbind-wechat` 和 `/api/v2/user/upload-avatar`

### 3.4 状态管理

- **全局状态**：用户信息、登录状态
- **组件状态**：表单数据、验证状态、加载状态
