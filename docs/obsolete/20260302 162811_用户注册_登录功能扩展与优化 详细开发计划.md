# 用户注册/登录功能扩展与优化 详细开发计划

## 一、项目背景与目标

### 1.1 项目背景

当前系统已实现基本的手机号注册/登录功能，但随着业务发展和用户需求的增长，需要扩展注册方式、丰富登录渠道、优化用户体验，以提升系统的竞争力和用户满意度。

### 1.2 开发目标

- **扩展注册方式**：新增邮箱+验证码注册，与现有手机+验证码形成双渠道
- **丰富登录渠道**：支持登录名/邮箱/手机+密码登录，以及微信扫码登录
- **优化用户体验**：区分登录名与昵称，设置合理的默认规则，提升操作流畅度
- **保障系统安全**：加强验证码安全、密码安全、敏感信息保护
- **确保兼容性**：兼容现有老用户，确保系统平滑过渡

## 三、各模块技术实现方案与接口设计

### 3.1 后端技术实现方案

#### 3.1.1 用户模型扩展

- **修改文件**：backend/app/models/user.py
- **新增字段**：
  - `login_name`：登录名，varchar(20)，唯一索引
  - `email`：邮箱，varchar(50)，唯一索引
  - `nickname`：昵称，varchar(20)，默认值为登录名
  - `avatar`：头像，varchar(255)，默认值为null
  - `wechat_openid`：微信OpenID，varchar(100)，唯一索引
  - `is_old_user`：是否老用户，tinyint(1)，默认值0

#### 3.1.2 验证码模块扩展

- **修改文件**：backend/app/utils/verify_code.py
- **新增功能**：
  - 支持邮箱验证码发送
  - 增加验证码类型区分（phone/email）
  - 实现验证码发送频率限制（60秒/次，单日5次）
  - 实现验证码有效时长控制（10分钟）

#### 3.1.3 用户API模块扩展

- **修改文件**：backend/app/api/user.py
- **新增接口**：
  1. **邮箱验证码发送接口**：`/api/v2/verify-code/email`
  2. **邮箱+验证码注册接口**：`/api/v2/register/email`
  3. **微信扫码注册接口**：`/api/v2/register/wechat`
  4. **登录名+密码登录接口**：`/api/v2/login/login-name`
  5. **邮箱+密码登录接口**：`/api/v2/login/email`
  6. **手机+密码登录接口**：`/api/v2/login/phone`
  7. **微信扫码登录接口**：`/api/v2/login/wechat`
  8. **老用户完善登录名接口**：`/api/v2/user/perfect-login-name`
  9. **昵称修改接口**：`/api/v2/user/update-nickname`
  10. **微信账号解绑接口**：`/api/v2/user/unbind-wechat`
  11. **头像上传接口**：`/api/v2/user/upload-avatar`

#### 3.1.4 认证与密码模块优化

- **修改文件**：backend/app/utils/token.py, backend/app/utils/password.py
- **优化内容**：
  - 增强JWT token生成与验证，支持多渠道登录信息
  - 优化密码加密算法（使用bcrypt）
  - 增加密码强度校验逻辑

#### 3.1.5 文件上传模块实现

- **修改文件**：backend/app/utils/file_upload.py
- **实现内容**：
  - 处理用户头像上传
  - 生成唯一文件名
  - 存储文件到指定目录
  - 返回文件URL

### 3.2 前端技术实现方案

#### 3.2.1 登录/注册页面优化

- **修改文件**：frontend/src/pages/Login/Login.jsx

## 六、潜在技术风险与业务风险评估及针对性应对策略

### 6.1 技术风险

| 风险描述 | 影响程度 | 应对策略 |
|---------|---------|----------|
| 数据库字段新增影响现有数据 | 高 | 采用非破坏性迁移，新增字段设置合理默认值，确保现有数据不丢失 |
| 验证码发送频率限制实现复杂 | 中 | 利用Redis实现分布式频率限制，确保多实例部署时的一致性 |
| 微信扫码登录集成复杂度高 | 中 | 参考微信官方文档，确保OpenID获取与绑定逻辑正确 |
| 老用户登录名完善流程设计复杂 | 中 | 设计合理的弹窗提示流程，确保老用户体验流畅 |

### 6.2 业务风险

| 风险描述 | 影响程度 | 应对策略 |
|---------|---------|----------|
| 注册流程变长导致用户流失 | 高 | 优化注册流程，减少必填项，确保流程简洁流畅 |
| 登录方式过多导致用户 confusion | 中 | 设计清晰的登录方式切换界面，提供明确的操作指引 |
| 密码强度要求过高导致用户体验下降 | 中 | 设置合理的密码强度要求，提供密码强度实时提示 |
| 敏感信息泄露风险 | 高 | 严格执行敏感信息加密存储和传输，接口返回隐藏敏感信息 |

## 八、与其他系统模块的集成方案

### 8.1 与现有用户系统的集成

- **用户数据同步**：确保新用户数据与现有用户数据结构一致，老用户数据平滑迁移
- **登录状态共享**：确保多渠道登录生成的token格式一致，可在系统其他模块通用
- **权限控制集成**：确保新的登录方式与现有权限控制体系兼容

### 8.2 与微信开放平台的集成

- **微信扫码登录流程**：
  1. 前端生成微信扫码二维码
  2. 用户扫码授权
  3. 前端获取微信OpenID
  4. 调用后端微信登录接口
  5. 后端验证OpenID并处理登录/注册逻辑

### 8.3 与邮箱服务的集成

- **邮箱验证码发送流程**：
  1. 前端调用邮箱验证码发送接口
  2. 后端生成验证码并存储
  3. 后端调用邮箱服务发送验证码
  4. 前端接收验证码并完成注册/登录
