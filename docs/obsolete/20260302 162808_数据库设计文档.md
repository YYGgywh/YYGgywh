# 用户注册/登录功能扩展与优化 数据库设计文档

## 一、数据库现状

### 1.1 现有数据库

当前系统使用SQLite数据库，数据库文件路径为：`backend/app/db/paipan.db`

### 1.2 现有用户表结构

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | 用户ID |
| phone | VARCHAR(11) | UNIQUE, NOT NULL, INDEX | 手机号 |
| password | VARCHAR(255) | NULL | 密码（哈希存储） |
| ext_info | TEXT | DEFAULT '{}' | 扩展信息（JSON格式） |
| create_time | INTEGER | DEFAULT CURRENT_TIMESTAMP | 创建时间戳 |
| update_time | INTEGER | DEFAULT CURRENT_TIMESTAMP | 更新时间戳 |
| is_active | BOOLEAN | DEFAULT TRUE | 是否激活 |

## 二、数据库扩展方案

### 2.1 用户表（user）扩展

| 字段名 | 数据类型 | 默认值 | 约束条件 | 说明 |
|-------|---------|--------|----------|------|
| login_name | VARCHAR(20) | NULL | UNIQUE, INDEX | 登录名，新用户必填，老用户可后续完善 |
| email | VARCHAR(50) | NULL | UNIQUE, INDEX | 用户邮箱，用于注册、密码找回 |
| nickname | VARCHAR(20) | login_name | NOT NULL | 用户昵称，默认等于登录名，可修改 |
| avatar | VARCHAR(255) | NULL | | 用户头像URL |
| wechat_openid | VARCHAR(100) | NULL | UNIQUE, INDEX | 微信OpenID，用于微信扫码登录 |
| is_old_user | TINYINT(1) | 0 | NOT NULL | 0=新用户，1=老用户，用于老用户完善登录名逻辑 |

### 2.2 验证码表（verify_code）设计

| 字段名 | 数据类型 | 默认值 | 约束条件 | 说明 |
|-------|---------|--------|----------|------|
| id | INTEGER | AUTOINCREMENT | PRIMARY KEY | 验证码ID |
| type | VARCHAR(10) | NOT NULL | | 验证码类型（phone=手机验证码，email=邮箱验证码） |
| phone | VARCHAR(11) | NULL | | 手机号（type=phone时必填） |
| email | VARCHAR(50) | NULL | | 邮箱（type=email时必填） |
| code | VARCHAR(6) | NOT NULL | | 验证码 |
| create_time | INTEGER | DEFAULT CURRENT_TIMESTAMP | | 创建时间 |
| expire_time | INTEGER | NOT NULL | | 过期时间 |
| send_count | INTEGER | 1 | NOT NULL | 发送次数 |

## 三、数据库迁移方案

### 3.1 迁移策略

采用非破坏性迁移方式，确保现有数据不丢失，同时新增所需字段。

### 3.2 迁移步骤

1. **备份现有数据库**：在迁移前备份paipan.db文件
2. **新增用户表字段**：使用ALTER TABLE语句添加新字段
3. **创建验证码表**：创建新的verify_code表
4. **更新老用户标记**：将现有用户标记为老用户（is_old_user=1）
5. **验证迁移结果**：确保现有数据正常，新字段可正常使用

### 3.3 迁移SQL语句

#### 3.3.1 新增用户表字段

```sql
-- 添加login_name字段
ALTER TABLE user ADD COLUMN login_name VARCHAR(20) NULL;
-- 添加email字段
ALTER TABLE user ADD COLUMN email VARCHAR(50) NULL;
-- 添加nickname字段
ALTER TABLE user ADD COLUMN nickname VARCHAR(20) NULL;
-- 添加avatar字段
ALTER TABLE user ADD COLUMN avatar VARCHAR(255) NULL;
-- 添加wechat_openid字段
ALTER TABLE user ADD COLUMN wechat_openid VARCHAR(100) NULL;
-- 添加is_old_user字段
ALTER TABLE user ADD COLUMN is_old_user TINYINT(1) DEFAULT 0 NOT NULL;

-- 创建唯一索引
CREATE UNIQUE INDEX IF NOT EXISTS idx_user_login_name ON user(login_name);
CREATE UNIQUE INDEX IF NOT EXISTS idx_user_email ON user(email);
CREATE UNIQUE INDEX IF NOT EXISTS idx_user_wechat_openid ON user(wechat_openid);
```

#### 3.3.2 创建验证码表

```sql
CREATE TABLE IF NOT EXISTS verify_code (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type VARCHAR(10) NOT NULL,
    phone VARCHAR(11) NULL,
    email VARCHAR(50) NULL,
    code VARCHAR(6) NOT NULL,
    create_time INTEGER DEFAULT CURRENT_TIMESTAMP,
    expire_time INTEGER NOT NULL,
    send_count INTEGER DEFAULT 1 NOT NULL
);

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_verify_code_phone ON verify_code(phone);
CREATE INDEX IF NOT EXISTS idx_verify_code_email ON verify_code(email);
CREATE INDEX IF NOT EXISTS idx_verify_code_expire_time ON verify_code(expire_time);
```

#### 3.3.3 更新老用户标记

```sql
UPDATE user SET is_old_user = 1;
```

## 四、数据兼容处理

### 4.1 老用户处理

- **登录名**：老用户首次登录时提示完善登录名
- **昵称**：老用户昵称默认设置为手机号
- **邮箱**：老用户邮箱默认为空，可后续绑定
- **微信OpenID**：老用户微信OpenID默认为空，可后续绑定

### 4.2 新用户处理

- **登录名**：新用户注册时必填，若未填写则自动生成
- **昵称**：新用户昵称默认等于登录名
- **邮箱**：新用户注册时可选（手机/邮箱二选一）
- **微信OpenID**：新用户通过微信注册时自动绑定

## 五、性能优化

### 5.1 索引优化

- 为login_name、email、wechat_openid字段创建唯一索引，提高查询效率
- 为verify_code表的phone、email字段创建索引，提高验证码查询效率

### 5.2 查询优化

- 使用索引覆盖查询，减少回表操作
- 合理使用JOIN操作，避免不必要的关联查询
- 定期清理过期验证码，保持表大小合理

## 六、安全考虑

### 6.1 数据加密

- 密码使用bcrypt算法加密存储
- 敏感信息（如手机号、邮箱）在接口返回时进行脱敏处理

### 6.2 数据验证

- 前端进行格式验证，后端进行二次验证
- 登录名、邮箱、微信OpenID确保唯一性
- 验证码有效期和发送频率控制

## 七、结论

本次数据库扩展方案采用非破坏性迁移方式，确保现有数据不丢失的同时，新增所需字段支持新功能。通过合理的索引设计和性能优化，确保数据库操作高效可靠。同时，通过安全措施保护用户数据，确保系统安全性。
