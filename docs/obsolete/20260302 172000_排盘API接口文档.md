# 排盘API接口文档

## 1. 接口概述

本文档详细描述了排盘系统的API接口，包括排盘记录的保存、查询、更新和删除等功能。

## 2. 接口列表

| 接口名称 | URL | 方法 | 功能描述 |
|---------|-----|------|----------|
| 保存排盘记录 | /api/pan/save | POST | 保存用户的排盘记录 |
| 查询排盘记录 | /api/pan/list | GET | 查询用户的排盘记录列表 |
| 更新排盘记录 | /api/pan/update/{record_id} | PUT | 更新排盘记录信息 |
| 删除排盘记录 | /api/pan/delete/{record_id} | DELETE | 删除排盘记录（逻辑删除） |

## 3. 接口详情

### 3.1 保存排盘记录

**URL**: `/api/pan/save`

**方法**: `POST`

**请求参数**:

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| pan_type | string | 否 | 排盘类型，默认值：liuyao |
| pan_params | string | 是 | 排盘参数，JSON格式字符串 |
| pan_result | string | 是 | 排盘结果，JSON格式字符串 |
| supplement | string | 否 | 求占者补充说明 |

**请求示例**:

```json
{
  "pan_type": "liuyao",
  "pan_params": "{\"method\": \"number\", \"number\": \"123\"}",
  "pan_result": "{\"gua\": \"乾卦\", \"yao\": \"初九\"}",
  "supplement": "测试排盘"
}
```

**响应示例**:

```json
{
  "code": 200,
  "msg": "保存成功",
  "data": {
    "record_id": 1
  }
}
```

### 3.2 查询排盘记录

**URL**: `/api/pan/list`

**方法**: `GET`

**请求参数**:

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| pan_type | string | 否 | 排盘类型，默认值：liuyao |
| page | integer | 否 | 页码，默认值：1 |
| size | integer | 否 | 每页数量，默认值：10 |
| start_time | integer | 否 | 开始时间戳 |
| end_time | integer | 否 | 结束时间戳 |

**响应示例**:

```json
{
  "code": 200,
  "msg": "查询成功",
  "data": [
    {
      "id": 1,
      "pan_params": {
        "method": "number",
        "number": "123"
      },
      "pan_result": {
        "gua": "乾卦",
        "yao": "初九"
      },
      "create_time": 1772441129,
      "update_time": 1772441129,
      "supplement": "测试排盘",
      "comment_count": 0
    }
  ]
}
```

### 3.3 更新排盘记录

**URL**: `/api/pan/update/{record_id}`

**方法**: `PUT`

**请求参数**:

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| record_id | integer | 是 | 排盘记录ID |
| supplement | string | 否 | 求占者补充说明 |
| audit_status | integer | 否 | 审核状态 |
| audit_remark | string | 否 | 审核备注 |

**请求示例**:

```json
{
  "supplement": "更新后的补充说明"
}
```

**响应示例**:

```json
{
  "code": 200,
  "msg": "更新成功",
  "data": {
    "record_id": 1
  }
}
```

### 3.4 删除排盘记录

**URL**: `/api/pan/delete/{record_id}`

**方法**: `DELETE`

**请求参数**:

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| record_id | integer | 是 | 排盘记录ID |

**响应示例**:

```json
{
  "code": 200,
  "msg": "删除成功",
  "data": {
    "record_id": 1
  }
}
```

## 4. 错误码说明

| 错误码 | 描述 |
|--------|------|
| 401 | 用户未认证 |
| 404 | 排盘记录不存在 |
| 500 | 服务器内部错误 |

## 5. 认证方式

所有接口都需要在请求头中添加 `Authorization` 字段，格式为 `Bearer {token}`，其中 `{token}` 是用户登录后获取的访问令牌。

## 6. 注意事项

1. 排盘参数和排盘结果必须是有效的JSON格式字符串
2. 删除操作是逻辑删除，不会真正删除数据库中的记录
3. 查询接口会自动过滤已删除的记录
4. 所有接口都有频率限制，请勿频繁调用
