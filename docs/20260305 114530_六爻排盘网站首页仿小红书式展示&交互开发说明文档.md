# 六爻排盘网站 小红书式展示&交互开发说明文档

# 文档前言

## 1.1 文档目的

本文档为六爻排盘网站「小红书式展示与交互」功能的开发落地指导文档，明确页面布局、组件设计、交互逻辑、技术实现、数据规范、性能要求等所有开发细节，为前端、后端开发人员提供统一、可执行的开发标准，确保开发成果贴合小红书核心交互风格，同时适配六爻排盘业务场景，保证功能可用、体验流畅、兼容性达标。

本文档为开发人员必备参考资料，开发过程中需严格遵循本文档规范，若有疑问或需调整，需提前沟通确认，避免偏离设计要求。

## 1.2 文档范围

本文档覆盖六爻排盘网站小红书式展示与交互的全部核心模块，具体包括：

- 整体页面布局（无规则瀑布流、响应式适配）；

- 核心组件（排盘内容卡片）的样式与交互；

- 卡片点击弹窗详情页的布局、内容与交互；

- 全局交互（滚动加载、导航吸顶、返回顶部等）；

- 技术实现要求、数据接口规范；

- 性能优化、兼容性要求；

- 开发注意事项与异常处理。

## 1.3 设计定位

本次开发以「复刻小红书核心展示与交互风格」为基础，结合六爻排盘网站的业务属性，实现“轻量化、沉浸式、实用性”的用户体验，核心定位如下：

- 展示模式：复刻小红书PC端无规则瀑布流+标准化内容卡片，移动端适配1列布局，聚焦排盘内容展示；

- 内容适配：将小红书的图文/视频替换为六爻排盘结果画像图+求测内容，保留卡片三层结构；

- 交互逻辑：复刻小红书轻量化交互（滚动加载、卡片hover、半屏弹窗），新增贴合六爻用户的收藏、图片放大、内容复制功能；

- 视觉风格：保持小红书简洁、低干扰的视觉特点，纯白背景、标准化字体、极简交互反馈，核心聚焦排盘业务内容。

## 1.4 适用人群

本文档适用于参与本次六爻排盘网站开发的所有相关人员，包括：前端开发工程师、后端开发工程师、测试工程师，其中前端开发人员需重点关注布局、组件、交互相关规范，后端开发人员需重点关注数据接口、字段规范、业务逻辑相关要求。

# 二、整体页面布局开发规范

## 2.1 核心布局：无规则瀑布流（PC端核心）

完全复刻小红书PC端无规则瀑布流逻辑，核心实现“列数固定+卡片高度自适应+自动补位+滚动触底无限加载”，适配六爻排盘卡片的内容特性，具体开发规范如下：

### 2.1.1 列数与响应式适配

采用CSS媒体查询实现不同屏幕尺寸的列数切换，核心断点明确，开发时需严格按照以下尺寸规范实现：

|屏幕宽度范围|列数|卡片宽度计算规则|页面边距|
|---|---|---|---|
|≥ 1200px（大屏PC）|3列（核心）|卡片宽度 =（页面可视宽度 - 左右边距总和 - 列间距×2）÷ 3|左右各20px，上下无固定边距|
|992px ≤ 宽度 ＜ 1200px（中屏PC）|2列|卡片宽度 =（页面可视宽度 - 左右边距总和 - 列间距×1）÷ 2|左右各20px，上下无固定边距|
|＜ 992px（平板/移动端）|1列|卡片宽度 = 页面可视宽度 - 左右边距总和|左右各10px，上下无固定边距|
### 2.1.2 间距与留白规范

间距统一，保证页面视觉平衡与呼吸感，开发时需严格遵循以下固定值：

- 列间距：固定15px（所有屏幕尺寸统一，不随屏幕宽度变化）；

- 行间距：固定15px（卡片上下间距，所有屏幕尺寸统一）；

- 卡片内边距：按卡片各区域规范执行，无额外外边距。

### 2.1.3 加载逻辑开发要求

加载逻辑需保证流畅性，无卡顿、无空白、无重复加载，具体开发要求：

1. 初始加载：页面首次渲染完成后，自动加载12张卡片（3列×4行），确保首屏视觉饱满，无空白区域；若后端返回数据不足12张，按实际数据展示，剩余位置不填充空白。

2. 滚动加载：监听页面滚动事件，当滚动条触底500px（即距离页面底部500px）时，自动触发下一批数据请求，每次加载6张卡片（3列×2行）；加载过程中，在瀑布流底部显示极简加载动画（3个灰色小圆点，直径8px，间隔4px，循环旋转），加载完成后动画自动消失，新卡片以淡入效果展示（opacity从0到1，过渡时间0.3s）。

3. 补位逻辑：新加载的卡片需自动填充到当前页面高度最低的列，保证页面整体视觉平衡，无大面积空白、无卡片错位；开发时需计算每列的当前高度，将新卡片插入到高度最小的列末尾。

4. 异常处理：若数据加载失败（接口报错、网络异常），在加载动画位置显示“加载失败，请重试”文字（样式与卡片内文字统一），点击文字可重新触发加载；若后端返回无更多数据，加载动画位置显示“已加载全部内容”，不再触发加载请求。

## 2.2 页面基础视觉规范

- 页面背景：纯#FFFFFF白色背景，无渐变、无图案、无水印，最大程度突出排盘卡片内容，禁止添加任何多余装饰。

- 顶部导航：复刻小红书悬浮吸顶式导航，开发要求如下：
        

    - 初始状态：导航栏背景透明，无边框，导航内容（logo、菜单）居中/左对齐展示；

    - 滚动状态：页面向下滚动≥50px时，导航栏背景变为#FFFFFF，添加1px浅灰色边框（#F5F5F5），导航内容保持不变；

    - 过渡效果：导航栏状态切换时，添加0.3s平滑过渡动画，无卡顿、无突兀感；

    - 层级：导航栏z-index设置为999，保证不被瀑布流卡片遮挡，同时不遮挡弹窗与蒙层。

- 冗余元素控制：页面无左侧/右侧固定侧边栏、无底部固定栏、无弹窗式广告，所有空间均用于瀑布流卡片展示，保持沉浸式浏览体验；禁止添加与核心功能无关的冗余元素。

## 2.3 空数据状态开发规范

当后端返回无排盘内容（列表为空）时，页面不显示瀑布流，需在页面居中位置展示空数据提示，具体规范：

- 提示内容：文字“暂无排盘内容，快来发布你的第一条排盘结果吧～”，搭配极简卦象图标（尺寸40px×40px，灰色#999999）；

- 样式：文字采用微软雅黑/苹方、字号14px、颜色#666666、行高1.5倍；图标与文字间距10px，整体水平+垂直居中；

- 交互：点击提示文字或图标，可跳转至排盘发布页面（若有），无其他额外交互。

# 三、核心组件：排盘内容卡片开发规范

排盘卡片为页面最小展示单元，完全复刻小红书圆角卡片+三层结构设计，将小红书的图文/视频替换为六爻排盘核心内容，卡片宽度随列数自适应，高度由内容自动计算，所有卡片样式标准化，仅内容不同，开发时需严格遵循以下规范：

## 3.1 卡片整体基础规范

- 形状与圆角：矩形卡片，四个角统一为12px圆角，无卡片边框，通过留白与页面背景区分，禁止添加阴影（hover时除外）。

- 尺寸规则：
        

    - 宽度：随瀑布流列数自适应，按2.1.1中的计算规则执行，无固定宽度；

    - 高度：无固定高度，由“排盘画像图区高度+求测内容区高度+用户/互动区高度”自动计算，保证内容完整展示。

- 视觉层级：从下到上分为三层，层间无间距，无缝衔接，整体视觉统一，层级顺序不可颠倒：
        

    1. 第一层：排盘结果画像图区（核心视觉区，占卡片高度70%-80%）；

    2. 第二层：求测内容标题区（文字区，占卡片高度10%-15%）；

    3. 第三层：用户+互动操作区（功能区，占卡片高度5%-10%）。

- 兼容性：卡片样式在所有现代浏览器中保持一致，无变形、无错位，适配不同屏幕尺寸的列数切换。

## 3.2 第一层：排盘结果画像图区（核心视觉区）

替换小红书的图片/视频区，为六爻排盘卡片的核心视觉部分，用户第一眼聚焦区域，开发规范如下：

- 内容载体：仅展示六爻排盘结果画像图（卦象图、排盘命盘图等），为用户排盘后生成的可视化图片，支持JPG/PNG格式，禁止展示其他类型内容。

- 图片尺寸与适配：
        

    - 图片宽度：与卡片宽度100%一致，无左右边距，铺满卡片宽度；

    - 图片高度：建议用户上传/系统生成的图片高度为300px-600px（区间内任意高度，保证瀑布流的无规则性）；

    - 图片裁剪：不强制裁剪，保持图片原始比例，若图片比例异常（过宽/过高），自动居中显示，无拉伸、无变形，超出卡片宽度/高度的部分隐藏（不滚动）。

- 视觉效果：图片无滤镜、无边框、无水印，底部与第二层求测内容区无缝衔接，无额外装饰；图片加载失败时，显示默认卦象占位图（灰色#F5F5F5背景，尺寸与卡片宽度一致，高度300px，居中显示“图片加载失败”文字）。

- 加载方式：采用懒加载模式，图片进入视口时才加载，提升页面首次加载速度；加载过程中，显示浅灰色占位背景（#F5F5F5），无加载动画。

## 3.3 第二层：求测内容标题区（文字区）

替换小红书的文案/标题区，展示用户六爻求测的核心内容，为卡片的文字核心，开发规范如下：

- 内容载体：仅展示用户六爻求测的问题/内容（如“问事业发展方向”“问感情复合可能性”），为用户发布排盘结果时填写的求测内容，禁止展示其他冗余文字。

- 文字样式（所有屏幕尺寸统一）：
        

    - 字体：微软雅黑/苹方（无衬线字体，保证跨设备兼容性）；

    - 字号：14px；

    - 字体颜色：#333333（深灰色，保证可读性）；

    - 行高：1.5倍；

    - 字重：常规（400）；

    - 禁止加粗、倾斜、下划线等额外样式。

- 文字展示规则：
        

    - 行数限制：最多展示2行，超出2行的部分用省略号（...）截断，激发用户点击卡片查看完整内容；

    - 对齐方式：左对齐；

    - 边距：左右各留10px内边距，上下各留8px内边距，无额外外边距；

    - 空内容处理：若用户未填写求测内容，显示默认文字“未填写求测内容”（样式与正常文字一致，颜色#999999）。

## 3.4 第三层：用户+互动操作区（功能区）

复刻小红书的“用户名+互动图标”布局，融合六爻网站的业务需求，仅保留用户名和点赞功能，无多余元素，开发规范如下：

- 整体布局：采用左-右横向布局，左侧展示发帖用户名，右侧展示点赞图标+点赞数，左右两端对齐，无中间元素，布局紧凑。

- 内边距：左右各留10px内边距，上下各留6px内边距，与第二层求测内容区无缝衔接，无间距。

- 左侧：发帖用户名
        

    - 内容载体：展示发布排盘结果的用户昵称（非登录名，为用户在网站的显示昵称）；

    - 文字样式：微软雅黑/苹方、字号12px、字体颜色#666666（中灰色）、行高1.2倍、字重常规；

    - 展示规则：单行展示，超出卡片宽度的部分用省略号（...）截断；若用户未设置昵称，显示默认昵称“六爻用户”。

- 右侧：点赞功能（复刻小红书红心图标）
        

    - 图标样式：完全复刻小红书红色心形图标，未点赞时为空心灰色（#999999），点赞后为实心红色（#FF2442），图标尺寸为16px×16px（所有屏幕尺寸统一），无额外边框；

    - 点赞数样式：紧邻点赞图标右侧，字体（微软雅黑/苹方）、字号12px、字体颜色#666666（中灰色），与图标间距4px；

    - 展示规则：点赞数≥0，若为0则仅显示图标，不显示数字；点赞数≥1000时，以“千”为单位简写（如1200→1.2k，2500→2.5k，保留1位小数）；

    - 交互规则：点击图标（无论是否包含数字）触发点赞/取消点赞操作，点击后图标状态实时切换，点赞数实时更新（无延迟，无弹窗反馈）；点赞状态与弹窗内点赞状态实时双向同步，与后端数据实时同步；

    - 异常处理：未登录用户点击点赞图标时，触发登录弹窗（仅包含登录入口与关闭按钮），不执行点赞操作；网络异常时，点击无响应，显示短暂提示“网络异常，请稍后再试”（灰色小弹窗，3秒后自动消失）。

## 3.5 卡片hover交互效果

为提升用户体验，卡片添加极简hover效果，无复杂动画，保持页面简洁，开发规范如下：

- hover状态：鼠标悬浮在卡片上时，卡片轻微上浮2px（transform: translateY(-2px)），同时添加浅灰色阴影（box-shadow: 0 2px 8px rgba(0,0,0,0.05)），无其他额外效果；

- 离开状态：鼠标离开卡片时，立即恢复原始状态（transform: translateY(0)，box-shadow: none），无过渡延迟；

- 一致性：所有卡片hover效果统一，无差异化；禁用状态（如异常卡片）无hover效果。

## 3.6 卡片点击交互

- 触发区域：点击排盘卡片除点赞图标外的任意区域，立即触发弹窗详情页（详见第四章），无延迟；点赞图标仅触发点赞操作，不触发弹窗。

- 点击反馈：点击卡片时，添加轻微的按压效果（transform: scale(0.98)），松开后恢复原状，过渡时间0.1s，提升交互手感；

- 异常处理：卡片加载未完成、数据异常时，点击无响应，鼠标指针变为“禁止”样式（cursor: not-allowed）。

# 四、核心交互：卡片点击弹窗详情页开发规范

复刻小红书「蒙层+右侧半屏弹窗」的卡片点击交互形式（PC端），移动端适配「蒙层+全屏弹窗」，替代传统页面跳转，保证用户浏览瀑布流的连贯性，弹窗内展示排盘结果的完整信息，兼顾信息完整性与交互流畅性，开发时需严格遵循以下规范：

## 4.1 弹窗整体基础规范（PC端核心）

- 弹窗触发：点击排盘卡片除点赞图标外的任意区域，立即触发弹窗，无延迟；点赞图标仅触发点赞操作，不触发弹窗；弹窗打开时，瀑布流页面禁止滚动。

- 整体结构：页面蒙层+右侧半屏弹窗，蒙层占满整个浏览器视口，弹窗固定在页面右侧，无滚动跟随，层级高于页面所有元素。

- 弹窗尺寸（所有PC屏幕尺寸统一）：
        

    - 宽度：固定600px，不随视口宽度变化；

    - 高度：占满浏览器视口100vh（顶部与导航栏齐平，底部与视口底部齐平）；

    - 圆角：仅弹窗左侧上下两个角为12px圆角，右侧无圆角，与小红书弹窗样式完全一致；

    - 边框：无边框，与蒙层形成视觉区分。

- 蒙层样式与交互：


    - 背景色：rgba(0,0,0,0.5)（半透明黑色），无渐变、无图案；

    - 交互：点击蒙层可关闭弹窗，恢复瀑布流页面原始状态（解除滚动禁止）；鼠标悬浮在蒙层上时，指针变为“指针”样式（cursor: pointer）；

    - 层级：蒙层z-index: 9998，低于弹窗，高于页面其他元素。

- 弹窗层级：弹窗z-index: 9999，高于蒙层与页面所有元素，保证无元素遮挡弹窗。

- 弹窗背景：纯#FFFFFF白色背景，与瀑布流页面背景一致，无渐变、无图案、无水印。

- 打开/关闭动画：
        

    - 打开：弹窗从页面右侧外沿向左侧滑入，动画时长0.3s，匀速过渡（transition: transform 0.3s linear）；蒙层同步从透明变为rgba(0,0,0,0.5)，过渡时长0.3s；

    - 关闭：弹窗向页面右侧外沿滑出，动画时长0.3s，匀速过渡；蒙层同步从rgba(0,0,0,0.5)变为透明，过渡时长0.3s；动画结束后，弹窗与蒙层销毁，释放DOM资源。

- 加载状态：点击卡片后，弹窗与蒙层立即显示，弹窗内显示极简加载动画（与瀑布流加载动画一致），同时异步请求排盘详情数据；数据加载完成后，动画消失，渲染弹窗内容；加载失败时，显示“加载失败，请重试”文字（居中显示，样式与卡片内文字一致），点击文字可重新请求。

## 4.2 弹窗移动端适配规范

移动端放弃半屏设计，改为蒙层+全屏弹窗，适配移动端触控操作，开发规范如下：

- 结构调整：蒙层+全屏弹窗，弹窗占满整个手机视口，无多余空白；

- 尺寸规则：宽度100vw，高度100vh，四个角均为12px圆角，无边框；

- 关闭方式：除点击蒙层可关闭弹窗外，弹窗顶部新增关闭按钮（×），位置在弹窗右上角，距离右侧15px、顶部15px，尺寸24px×24px，颜色#666666，hover时变为#333333；

- 动画调整：打开/关闭动画改为从下到上滑入/滑出，动画时长0.3s，匀速过渡，贴合移动端用户操作习惯；

- 交互适配：弹窗内所有可点击元素（按钮、图标）的触控区域≥40px×40px，避免触控误操作；滚动区域适配移动端触控滚动，无卡顿。

## 4.3 PC端弹窗内部布局与内容分区

弹窗内部按从上到下的顺序分为五大固定区域，区域间无间距，无缝衔接，整体为单栏滚动布局（仅弹窗内容区可滚动，弹窗主体固定），开发时需严格遵循分区顺序与布局规范，分区如下：

1. 顶部：弹窗关闭区（固定，不随滚动变化）；

2. 区域1：排盘核心视觉区（排盘画像大图，固定宽度，高度自适应）；

3. 区域2：排盘基础信息区（求测内容+发布用户信息，核心文字区）；

4. 区域3：排盘详细解析区（卦象解析、爻辞解读等，核心业务区）；

5. 区域4：弹窗互动操作区（点赞+收藏，固定在弹窗底部，不随滚动变化）。

核心规则：仅区域3（排盘详细解析区）可根据内容高度纵向滚动，其余区域均为固定定位，不随滚动变化，保证用户操作的便捷性；滚动时，区域3的滚动条为浏览器原生样式，做浅化处理（颜色#F5F5F5，宽度6px），不突兀。

## 4.4 弹窗各区域详细开发规范

### 4.4.1 顶部：弹窗关闭区

- 布局：弹窗最顶部，宽度100%（600px），高度60px，水平居中，垂直居中对齐；

- 内容：仅展示关闭按钮（×），无其他冗余元素；

- 按钮样式：
        

    - 形状：方形，尺寸24px×24px；

    - 颜色：#666666（中灰色），hover时变为#333333（深灰色），无背景色、无边框；

    - 位置：弹窗右上角，距离弹窗右侧20px，顶部20px；

    - 字体：微软雅黑/苹方，字号20px，居中显示。

- 交互：点击按钮立即关闭弹窗，触发关闭动画，与点击蒙层效果一致；点击时添加轻微按压反馈（transform: scale(0.9)），松开后恢复原状；

- 固定规则：该区域为固定定位（position: fixed），top: 0，right: 0，width: 600px，z-index: 10000，不随弹窗内容滚动变化，始终显示在弹窗顶部。

### 4.4.2 区域1：排盘核心视觉区

- 布局：关闭区下方，宽度100%（600px），无左右边距，顶部与关闭区无缝衔接；

- 内容：展示排盘结果画像大图（为卡片中画像图的高清版本），支持JPG/PNG格式，禁止展示其他内容；

- 图片适配：
        

    - 宽度：与弹窗宽度100%一致（600px），无左右边距；

    - 高度：根据图片原始比例自适应，无固定高度，保证图片清晰度，无拉伸、无裁剪；

    - 对齐：水平居中，垂直顶对齐；

    - 加载失败：显示默认卦象占位图（灰色#F5F5F5背景，尺寸600px宽，高度300px，居中显示“图片加载失败”文字）。

- 视觉效果：图片无滤镜、无边框、无水印，底部与区域2无缝衔接；

- 加载方式：采用预加载模式，点击卡片时提前加载高清图，加载过程中显示极简加载动画（与弹窗加载动画一致），避免弹窗内图片加载空白；

- 交互：点击图片可触发图片全屏放大预览，支持鼠标滚轮缩放（缩放范围1-2倍）、点击空白处关闭预览；预览时，蒙层背景变为rgba(0,0,0,0.8)，图片居中显示，无其他冗余元素。

### 4.4.3 区域2：排盘基础信息区

- 布局：视觉区下方，宽度100%，左右各留30px内边距，上下各留20px内边距，与视觉区无缝衔接；

- 内容分区：从上到下分为求测内容模块和发布用户模块，模块间距15px，无其他冗余元素；

- 求测内容模块：
        

    - 内容：展示用户完整的六爻求测内容（无行数限制，区别于卡片的2行截断）；

    - 文字样式：微软雅黑/苹方、字号18px、字体颜色#333333、行高1.6倍、字重500（中等加粗）；

    - 对齐：左对齐；

    - 空内容处理：若用户未填写求测内容，显示默认文字“未填写求测内容”（样式与正常文字一致，颜色#999999）。

- 发布用户模块：
        

    - 布局：左-右横向布局，左侧为用户名，右侧为发布时间；

    - 用户名：微软雅黑/苹方、字号14px、字体颜色#666666、字重常规；展示完整昵称，无截断；若用户未设置昵称，显示默认昵称“六爻用户”；

    - 发布时间：微软雅黑/苹方、字号12px、字体颜色#999999、字重常规；格式为YYYY-MM-DD HH:MM（如2026-03-05 14:30），禁止其他格式；

    - 间距：两个元素间距10px，右对齐靠弹窗右侧内边距；

    - 交互：点击用户名可跳转至用户个人中心（若有），无其他交互。

### 4.4.4 区域3：排盘详细解析区

该区域为弹窗核心业务区，展示排盘的完整解析内容，开发规范如下：

- 布局：基础信息区下方，宽度100%，左右各留30px内边距，顶部留10px内边距，底部与弹窗互动区无缝衔接；

- 内容载体：展示排盘的完整业务解析内容，包含但不限于：卦象名称、爻辞原文、卦象解读、吉凶判断、建议分析等，具体内容由后端接口返回；

- 文字样式规范（所有内容严格遵循）：
       

    - 一级标题（如卦象名称）：微软雅黑/苹方、字号16px、颜色#333333、行高1.5倍、字重600、上下间距10px；

    - 二级标题（如爻辞原文/卦象解读）：微软雅黑/苹方、字号14px、颜色#444444、行高1.5倍、字重500、上下间距8px；

    - 正文内容（如解读文字/建议）：微软雅黑/苹方、字号14px、颜色#666666、行高1.6倍、字重常规、段落间距10px；

    - 禁止使用任何彩色文字、加粗以外的特殊样式，保持文字简洁、易读。

- 排版规则：
        

    - 所有内容左对齐，无首行缩进，段间距清晰；

    - 不同解析模块（如卦象解读、建议分析）之间，添加浅灰色分隔线（height:1px; background:#F5F5F5;），分隔线宽度与解析区宽度一致（无左右边距），提升可读性；

    - 若解析内容为空，显示默认文字“暂无排盘解析内容”（居中显示，样式与正文内容一致，颜色#999999）。

- 滚动规则：该区域为弹窗唯一可滚动区域，当解析内容高度超过弹窗剩余视口高度时，自动出现纵向滚动条；滚动条样式做浅化处理（颜色#F5F5F5，宽度6px），滚动时无卡顿、无掉帧；滚动过程中，其他区域（关闭区、视觉区、互动区）保持固定，不随滚动变化。

- 交互：解析区所有文字内容支持鼠标选中复制，无禁止复制限制，贴合六爻用户保存解析内容的需求；禁止添加其他额外交互。

- 内容：仅包含点赞功能和收藏功能，左右横向布局，水平居中排列，无其他冗余元素；

- 元素样式与间距：
        

    - 两个功能模块间距50px；

    - 每个模块由图标+文字组成，图标在上，文字在下，垂直居中，模块内部图标与文字间距5px；

    - 整体垂直居中：两个模块在70px高度内垂直居中对齐。

- 点赞模块（与卡片同步）：
        

    - 图标：与卡片点赞图标一致，空心灰色#999999/实心红色#FF2442，尺寸24px×24px，无额外边框；

    - 文字：“点赞”，微软雅黑/苹方、字号12px、颜色与图标一致，字重常规；

    - 点赞数：展示在文字右侧，与文字间距4px，样式与文字一致；点赞数≥0，0则不显示数字，≥1000则以“千”为单位简写；

    - 交互：点击图标/文字均触发点赞/取消点赞，与瀑布流卡片的点赞状态实时双向同步，与后端数据实时同步；点击时添加轻微按压反馈（transform: scale(0.9)），松开后恢复原状；未登录用户点击时，触发登录弹窗，不执行点赞操作；网络异常时，点击无响应，显示短暂提示。

- 收藏模块（新增核心功能）：

    - 图标：书签样式，未收藏时为空心灰色#999999，收藏后为实心橙色#FF9F1C，尺寸24px×24px，无额外边框；

    - 文字：“收藏”，微软雅黑/苹方、字号12px、颜色与图标一致，字重常规；

    - 收藏数：展示在文字右侧，与文字间距4px，样式与文字一致；收藏数≥0，0则不显示数字，≥1000则以“千”为单位简写；

    - 交互：点击图标/文字均触发收藏/取消收藏，状态实时切换，与后端数据实时同步；点击时添加轻微按压反馈（transform: scale(0.9)），松开后恢复原状；未登录用户点击时，触发登录弹窗，不执行收藏操作；网络异常时，点击无响应，显示短暂提示；收藏状态与后端实时同步，关闭弹窗后再次打开，状态不丢失。

- 固定规则：该区域为固定定位（position: fixed），bottom: 0，right: 0，width: 600px，z-index: 10000，不随弹窗内容滚动变化，始终显示在弹窗底部。

## 4.5 弹窗内额外交互与异常处理

### 4.5.1 额外交互

- 图片放大：点击排盘核心视觉区的高清画像图，触发图片全屏放大预览，支持鼠标滚轮缩放（1-2倍）、点击空白处/ESC键关闭预览；预览时，蒙层背景变为rgba(0,0,0,0.8)，图片居中显示，无其他冗余元素；

- 内容复制：排盘详细解析区的所有文字内容支持鼠标选中复制，无禁止复制限制；复制后无弹窗反馈，保持交互轻量化；

- 键盘操作：支持ESC键关闭弹窗，关闭时触发关闭动画，恢复瀑布流页面滚动；

- 无二级弹窗：弹窗内所有操作（点赞、收藏、图片放大）均无二级弹窗，仅做状态实时切换，保持交互轻量化。

### 4.5.2 异常处理

- 数据加载失败：弹窗打开后，若详情接口请求失败，显示“加载失败，请重试”文字（居中显示，样式与卡片内文字一致），点击文字可重新请求；

- 网络异常：点击点赞/收藏按钮时，若网络异常，无状态变化，显示短暂提示“网络异常，请稍后再试”（灰色小弹窗，3秒后自动消失）；

- 未登录操作：未登录用户点击点赞/收藏按钮时，触发登录弹窗（仅包含登录入口与关闭按钮），登录弹窗样式与本文档视觉规范一致，不影响弹窗详情页的展示；

- 数据异常：若后端返回的解析内容、图片URL等数据异常，显示对应默认内容（如默认图片、默认文字），不报错、不影响弹窗正常显示。

# 五、全局交互开发规范

## 5.1 瀑布流核心交互

- 滚动交互：支持鼠标滚轮、页面滚动条、触控屏滑动等多种滚动方式，滚动过程中卡片无卡顿、无错位，加载新卡片时页面无抖动；滚动速度适配用户操作，无过快/过慢现象；

- 加载交互：滚动触底加载时，仅在瀑布流底部显示极简加载动画（3个灰色小圆点，直径8px，间隔4px，循环旋转），加载完成后动画自动消失，新卡片淡入展示（opacity从0到1，过渡时间0.3s）；加载失败时，显示“加载失败，请重试”，点击可重新加载；无更多数据时，显示“已加载全部内容”；

- 空数据交互：空数据状态下，点击提示文字/图标可跳转至排盘发布页面（若有），无其他交互。

## 5.2 页面全局交互

### 5.2.1 导航栏交互

复刻小红书悬浮吸顶交互，开发规范如下：

- 初始状态：导航栏背景透明，无边框，导航内容（logo、菜单）正常展示；

- 滚动状态：页面向下滚动≥50px时，导航栏背景变为#FFFFFF，添加1px浅灰色边框（#F5F5F5），导航内容保持不变；

- 过渡效果：导航栏状态切换时，添加0.3s平滑过渡动画（transition: background-color 0.3s, box-shadow 0.3s），无卡顿、无突兀感；

- 交互：导航栏菜单点击可跳转至对应页面（如首页、排盘页、个人中心），点击logo可跳转至首页；导航栏始终固定在页面顶部，不随滚动消失。

### 5.2.2 返回顶部交互

- 显示条件：当页面向下滚动≥500px时，在页面右下角显示返回顶部按钮；页面滚动到顶部时，按钮自动隐藏；

- 按钮样式：圆形，直径40px，白色背景（#FFFFFF），添加轻微阴影（box-shadow: 0 2px 8px rgba(0,0,0,0.1)），内部为灰色向上箭头（尺寸20px×20px，颜色#666666）；

- 位置：页面右下角，距离右侧20px，距离底部20px，固定定位（position: fixed），z-index: 999；

- 交互：点击按钮，页面平滑滚动回顶部，滚动时间0.5s（transition: scroll-behavior 0.5s）；鼠标hover时，按钮背景变为#F5F5F5，无其他效果；

- 兼容性：按钮在所有现代浏览器中正常显示，无变形、无错位，触控操作正常。

### 5.2.3 登录引导交互

页面无强制登录、无强制弹窗、无强制引导，开发规范如下：

- 游客权限：游客可正常浏览所有排盘卡片和弹窗详情，可查看完整解析内容、复制文字、放大图片；

- 登录引导：仅在用户点击点赞、收藏、发布排盘等操作时，触发登录弹窗；

- 登录弹窗样式：弹窗居中显示，宽度300px，高度200px，白色背景，12px圆角，无多余装饰；包含手机登录、微信登录两个入口，底部有关闭按钮；文字样式与本文档规范一致；

- 交互：登录弹窗可关闭（点击关闭按钮或蒙层），关闭后返回原页面，不影响用户继续浏览；登录成功后，自动关闭弹窗，恢复用户之前的操作（如点赞、收藏）。

# 六、技术实现要求

## 6.1 前端技术实现要求

前端开发需基于原生JS/JS框架（Vue/React）实现，优先保证功能稳定性、交互流畅性、兼容性，具体要求如下：
