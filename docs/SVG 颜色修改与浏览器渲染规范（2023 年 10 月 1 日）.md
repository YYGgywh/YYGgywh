# SVG 颜色修改与浏览器渲染规范

## 一、文档说明

本文档明确项目中SVG图片在浏览器端的颜色修改方式、引入规范及CSS样式适配规则，统一开发标准，保证SVG样式修改的一致性和可维护性。

## 二、核心原则

SVG颜色支持通过CSS样式直接修改，**修改生效的关键为SVG的引入方式**及**内部元素未硬编码颜色**，项目中优先采用支持CSS改色的引入方式，避免使用隔离样式的引入方式。

## 三、SVG 引入方式规范

### 3.1 推荐使用（支持CSS改色）

此类方式下浏览器将SVG解析为DOM元素，外部CSS可直接选中内部标签修改样式，按优先级排序：

1. **SVG代码内嵌至HTML/JSX**：改色灵活性最高，为项目首选方式；

2. **<object>/<iframe>标签引入独立SVG文件**：适合需单独管理的SVG资源，注意处理跨域问题。

### 3.2 禁止使用（不支持CSS改色）

此类方式下SVG被当作图片资源解析，内部样式被浏览器隔离，外部CSS无法穿透修改颜色：

1. `<img src="xxx.svg">`标签引入；

2. CSS中通过`background-image: url(xxx.svg)`设置为背景图；

3. JS中通过`new Image()`加载SVG资源。

## 四、CSS 修改 SVG 颜色方法

### 4.1 通用前提

SVG内部的填充色（`fill`）、描边色（`stroke`）**不可硬编码具体颜色值**（如`fill="#FFD700"`），需设为`currentColor`/`inherit`，或直接删除`fill`/`stroke`属性，为CSS留修改入口。

### 4.2 方法一：currentColor 统色法（推荐）

1. SVG内部将`fill`/`stroke`属性值设为`currentColor`，实现颜色占位；

2. 外部CSS通过控制`color`属性，一键修改SVG的填充和描边颜色，适配组件正常/禁用态等场景，代码示例：

```XML

<!-- 内嵌SVG代码：颜色占位 -->
<svg class="svg-icon" width="100" height="100" viewBox="0 0 100 100">
  <path d="M20,80 C40,20 60,20 80,80 Z" fill="currentColor" stroke="currentColor" stroke-width="2"/>
</svg>
```

```CSS

/* CSS Modules 中样式修改 */
/* 正常态颜色 */
.svg-icon {
  color: #FFD700; /* 金色 */
}
/* 禁用态颜色 */
.svg-icon.disabled {
  color: #CCCCCC; /* 淡灰色，适配项目控件禁用态配色 */
}
```

### 4.3 方法二：元素精准改色法

外部CSS直接选中SVG内部的`<path>`/`<circle>`/`<rect>`等标签，单独修改`fill`（填充色）、`stroke`（描边色），适合需区分填充/描边颜色的场景，代码示例：

```CSS

/* 正常态：填充金色，描边暗金色 */
.svg-icon path {
  fill: #FFD700;
  stroke: #B8860B;
}
/* 禁用态：适配项目禁用配色 */
.svg-icon.disabled path {
  fill: #F5F5F5;
  stroke: #CCCCCC;
}
```

## 五、注意事项

1. **样式优先级**：外部CSS样式 > SVG内部`style`属性 > SVG内部`fill`/`stroke`属性，开发时避免在SVG内部写`style`属性覆盖样式；

2. **独立SVG文件改色**：通过`<object>`引入的独立SVG文件，需在文件内部将`fill`/`stroke`设为`inherit`，或通过JS操作`<object>`的DOM内容修改颜色；

3. **CSS Modules 兼容**：内嵌SVG的改色样式可直接写在项目`.module.css`文件中，通过类名选中即可，无兼容性问题；

4. **全局样式适配**：项目通用SVG图标颜色可在全局样式文件中定义，组件内按需重写即可。

## 六、项目适配要求

1. 所有组件内的SVG图标，均采用**内嵌式**引入，统一使用`currentColor`占位法做颜色控制；

2. SVG图标需适配组件禁用态时，在组件CSS中通过`.disabled`类修改`color`属性即可，与项目控件禁用态配色保持一致；

3. 禁止在SVG代码中硬编码颜色值，若需引入第三方SVG文件，需先编辑文件删除硬编码的`fill`/`stroke`颜色值后再使用。
