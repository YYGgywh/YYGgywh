### 2.3 核心组件设计

#### 2.3.1 瀑布流容器组件 (WaterfallContainer)

**功能：**
- 实现瀑布流布局计算
- 管理卡片数据加载
- 处理滚动加载逻辑
- 响应式布局适配

**关键属性：**
- `items`：卡片数据数组
- `loading`：加载状态
- `hasMore`：是否有更多数据
- `onLoadMore`：加载更多回调
- `onCardClick`：卡片点击回调
- `onLikeToggle`：点赞切换回调

**核心逻辑：**
- 使用`useEffect`监听窗口大小变化，重新计算布局
- 使用`useInfiniteScroll`实现滚动加载
- 使用`waterfallLayout.js`计算卡片位置

#### 2.3.2 瀑布流卡片组件 (WaterfallCard)

**功能：**
- 展示排盘结果卡片
- 处理图片懒加载
- 实现点赞功能
- 提供hover效果

**关键属性：**
- `data`：卡片数据
- `onClick`：点击回调
- `onLikeToggle`：点赞切换回调

**核心逻辑：**
- 使用`IntersectionObserver`实现图片懒加载
- 处理图片加载失败情况
- 实现点赞状态切换

#### 2.3.3 弹窗详情组件 (PanDetailModal)

**功能：**
- 展示排盘详细信息
- 实现图片放大预览
- 处理点赞/收藏功能
- 支持键盘操作

**关键属性：**
- `isOpen`：是否打开
- `onClose`：关闭回调
- `data`：排盘详情数据
- `onLikeToggle`：点赞切换回调
- `onCollectToggle`：收藏切换回调

**核心逻辑：**
- 监听ESC键关闭弹窗
- 实现图片放大预览
- 同步点赞/收藏状态

### 2.4 状态管理

使用React Context进行状态管理，主要管理以下状态：

**WaterfallContext：**
- `cards`：卡片数据数组
- `loading`：加载状态
- `hasMore`：是否有更多数据
- `selectedCard`：当前选中的卡片
- `isModalOpen`：弹窗是否打开
- `loadMore`：加载更多方法
- `openModal`：打开弹窗方法
- `closeModal`：关闭弹窗方法

**UserContext：**
- `user`：当前用户信息
- `isAuthenticated`：是否已登录
- `login`：登录方法
- `logout`：登出方法

## 五、前后端交互设计

### 5.1 数据流转流程

**1. 首页瀑布流加载：**
1. 前端访问首页 `/`
2. 前端调用 `/api/v1/pan/public/list` 获取初始12条排盘记录
3. 后端查询数据库，返回排盘记录列表
4. 前端渲染瀑布流布局
5. 前端监听滚动事件，触底时调用 `/api/v1/pan/public/list` 加载更多

**2. 卡片点击查看详情：**
1. 用户点击卡片
2. 前端调用 `/api/v1/pan/detail/{id}` 获取排盘详情
3. 后端查询数据库，返回排盘详情
4. 前端打开弹窗，展示详情

**3. 点赞/收藏操作：**
1. 用户点击点赞/收藏按钮
2. 前端检查用户是否登录，未登录则显示登录弹窗
3. 已登录用户，前端调用 `/api/v1/pan/like/toggle` 或 `/api/v1/pan/collect/toggle`
4. 后端处理点赞/收藏逻辑，更新数据库
5. 前端更新UI状态

### 5.2 响应格式设计

**成功响应：**
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {...}
}
```

**错误响应：**
```json
{
  "code": 400,
  "msg": "参数错误",
  "data": null
}
```

### 5.3 异常处理

**前端异常处理：**
- API请求失败：显示错误提示，提供重试按钮
- 图片加载失败：显示默认占位图
- 网络异常：显示网络错误提示

**后端异常处理：**
- 参数验证错误：返回400状态码
- 未认证错误：返回401状态码
- 权限错误：返回403状态码
- 资源不存在：返回404状态码
- 服务器内部错误：返回500状态码

## 六、性能优化方案

### 6.1 前端性能优化

**1. 图片优化：**
- 图片懒加载：使用Intersection Observer实现
- 图片压缩：服务器端压缩图片
- 图片格式：使用WebP格式，提供fallback

**2. 渲染优化：**
- 组件懒加载：使用React.lazy和Suspense
- 虚拟滚动：只渲染可视区域内的卡片
- 减少重绘：使用CSS transform代替top/left

**3. 网络优化：**
- HTTP/2：启用HTTP/2
- 缓存策略：合理设置缓存头
- 数据缓存：本地存储常用数据

### 6.2 后端性能优化

**1. 数据库优化：**
- 索引优化：添加合适的索引
- 查询优化：减少不必要的字段查询
- 分页查询：使用LIMIT和OFFSET

**2. API优化：**
- 响应缓存：使用Redis缓存热点数据
- 批量请求：合并多个API请求
- 异步处理：使用FastAPI的异步特性

**3. 服务器优化：**
- Gunicorn：使用Gunicorn作为WSGI服务器
- 多进程：开启多个工作进程
- 负载均衡：配置负载均衡

### 6.3 监控与调优

**1. 性能监控：**
- 前端：使用Lighthouse、Chrome DevTools
- 后端：使用FastAPI的内置监控
- 数据库：使用SQLite性能分析工具

**2. 日志系统：**
- 前端：使用console.log和Sentry
- 后端：使用Python logging模块
- 错误追踪：集成Sentry

## 七、安全性设计

### 7.1 前端安全

**1. XSS防护：**
- 输入验证：对用户输入进行验证
- 输出编码：对输出进行HTML编码
- CSP：设置Content-Security-Policy

**2. CSRF防护：**
- CSRF Token：使用CSRF Token
- SameSite Cookie：设置SameSite=strict

**3. 敏感信息保护：**
- 不存储敏感信息：不在localStorage存储敏感信息
- HTTPS：使用HTTPS传输

### 7.2 后端安全

**1. 认证与授权：**
- JWT：使用JWT进行认证
- 权限控制：基于角色的权限控制
- 密码哈希：使用bcrypt哈希密码

**2. 输入验证：**
- Pydantic：使用Pydantic进行数据验证
- 参数检查：检查所有输入参数
- SQL注入防护：使用ORM防止SQL注入

**3. API安全：**
- 限流：实现API限流
- 速率限制：限制单位时间内的请求次数
- 防爬虫：识别并阻止恶意爬虫

### 7.3 部署安全
