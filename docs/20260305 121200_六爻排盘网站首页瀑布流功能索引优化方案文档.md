## 一、索引优化概述

### 1.1 索引的作用

- **加速查询**：通过索引快速定位数据，减少扫描范围
- **提高排序效率**：索引本身是有序的，可直接用于排序
- **唯一性保证**：通过唯一索引确保数据唯一性
- **外键约束**：加速外键关联查询

### 1.2 索引的类型

- **主键索引**：唯一且非空，表的主键自动创建
- **唯一索引**：确保字段值唯一
- **普通索引**：加速查询，不保证唯一性
- **复合索引**：多个字段组合的索引
- **覆盖索引**：包含查询所需的所有字段

### 1.3 索引的开销

- **存储空间**：索引需要额外的存储空间
- **写入性能**：插入、更新、删除操作需要维护索引
- **维护成本**：索引需要定期重建和优化

**优化说明**：

- 使用覆盖索引，避免回表查询

#### 4.1.2 排盘详情查询优化

**原始查询**：
```sql
SELECT * FROM pan_record 
WHERE id = ? AND is_visible = 1 AND deleted_at IS NULL;
```

**优化后查询**：
```sql

```

**优化说明**：
- 利用主键索引，直接定位数据
- 减少不必要的条件判断

#### 4.1.3 点赞/收藏状态查询优化

**原始查询**：
```sql

WHERE user_id = ? AND pan_id = ?;
```

**优化后查询**：
```sql
SELECT 1 FROM pan_like 
WHERE user_id = ? AND pan_id = ? 
LIMIT 1;
```

**优化说明**：
- 使用LIMIT 1，找到第一条匹配记录后立即返回
- 减少计数操作的开销

### 4.2 缓存策略

1. **Redis缓存**：
   - 缓存热点排盘记录
   - 缓存用户点赞/收藏状态
   - 缓存统计数据

2. **本地缓存**：
   - 前端缓存最近浏览的排盘记录
   - 后端服务缓存常用查询结果

3. **缓存失效策略**：
   - 点赞/收藏操作时更新缓存
   - 排盘记录更新时清除相关缓存
   - 设置合理的缓存过期时间

### 4.3 分页优化

1. **使用LIMIT和OFFSET**：
   - 避免一次性查询大量数据
   - 合理设置页大小，建议10-20条

2. **游标分页**：
   - 使用上次查询的最后一条记录作为游标
   - 避免OFFSET的性能问题

3. **预加载**：
   - 提前加载下一页数据
   - 提升用户体验
