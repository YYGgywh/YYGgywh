## 二、已注册用户处理方案

### 2.1 处理原则

1. **分级处理**：根据敏感词级别采取不同的处理措施
2. **用户权益保护**：在确保安全的前提下，尽量减少对用户的影响
3. **渐进式处理**：给予用户整改期限，避免一刀切
4. **申诉机制**：建立用户申诉渠道，处理误判情况

### 2.2 禁止级别敏感词用户处理方案

#### 2.2.1 处理措施

| 措施 | 说明 | 执行时间 |
|------|------|----------|
| **强制重置昵称** | 立即将昵称重置为临时名称（如：用户_xxx） | T+0 |
| **强制重置登录名** | 立即将登录名重置为系统生成名称（如：YYG_xxxxxxxx） | T+0 |
| **发送通知** | 通过短信/邮件通知用户昵称/登录名已被重置 | T+0 |
| **登录提示** | 用户登录时显示提示，要求修改昵称/登录名 | T+0 |
| **功能限制** | 限制部分功能（如评论、发布内容）直到修改完成 | T+7 |

#### 2.2.2 处理流程

```
Day 0 (立即执行)
├── 识别包含禁止级别敏感词的用户
├── 强制重置昵称和登录名
├── 发送系统通知
└── 记录操作日志

Day 1-7 (观察期)
├── 监控用户登录情况
├── 跟踪用户是否修改昵称/登录名
└── 提供修改指导

Day 7+ (限制期)
├── 对未修改的用户限制部分功能
├── 再次发送提醒通知
└── 记录违规次数
```


```
├── 用户可正常登录和使用系统
├── 登录时显示修改提示
├── 提供修改入口和指导
└── 第15天发送第二次提醒

Day 30-60 (限制期)
├── 限制部分功能（评论、发布内容等）
├── 发送第三次提醒通知
└── 强调修改的必要性

Day 60+ (强制期)
├── 强制重置昵称/登录名
├── 发送强制重置通知
└── 用户可重新修改
```

## 三、敏感词级别管理机制

### 3.1 当前敏感词级别定义

| 级别 | 名称 | 说明 | 处理方式 |
|------|------|------|----------|
| Level 2 | 禁止级别 | 系统保留词、严重违规词 | 完全禁止，拒绝通过 |
| Level 1 | 警告级别 | 一般敏感词、测试相关词 | 允许通过，但提示警告 |

### 3.2 警告级别敏感词注册处理逻辑

#### 3.2.1 当前实现

```python
# 注册时检测逻辑
is_safe, found_words = filter.check(login_name)

if not is_safe:
    forbidden_words = [w for w in found_words if w['level'] == 2]
    if forbidden_words:
        # 只有禁止级别敏感词才拒绝注册
        word_list = ", ".join([w['word'] for w in forbidden_words])
        raise HTTPException(
            status_code=400, 
            detail=f"登录名包含禁止使用的敏感词：{word_list}，请修改后重试"
        )
    # 警告级别敏感词允许注册，不提示
```

#### 3.2.2 优化建议

**方案A：注册时提示（推荐）**

```python
# 优化后的注册逻辑
is_safe, found_words = filter.check(login_name)

if not is_safe:
    forbidden_words = [w for w in found_words if w['level'] == 2]
    warning_words = [w for w in found_words if w['level'] == 1]
    
    if forbidden_words:
        # 禁止级别敏感词 - 拒绝注册
        word_list = ", ".join([w['word'] for w in forbidden_words])
        raise HTTPException(
            status_code=400, 
            detail=f"登录名包含禁止使用的敏感词：{word_list}，请修改后重试"

    if warning_words:
        # 警告级别敏感词 - 允许注册，但记录并提示
        word_list = ", ".join([w['word'] for w in warning_words])
        # 在响应中添加警告信息
```

### 3.3 敏感词级别升级处理流程

#### 3.3.1 升级场景

当管理员将某个敏感词从**警告级别（Level 1）**提升为**禁止级别（Level 2）**时，需要处理以下情况：

1. **已注册用户**：昵称/登录名包含该敏感词
2. **历史数据**：评论、内容中包含该敏感词
3. **新注册用户**：需要拦截包含该敏感词的注册

#### 3.3.2 升级处理流程

```
敏感词级别升级
    ↓
触发全量扫描任务
    ↓
├─ 扫描用户数据
│   ├─ 昵称包含敏感词的用户
│   ├─ 登录名包含敏感词的用户
│   └─ 生成用户清单
│
├─ 扫描历史数据
│   ├─ 评论内容包含敏感词的记录
│   ├─ 其他用户生成内容
│   └─ 生成内容清单
│
└─ 执行处理措施
    ├─ 用户数据：按禁止级别处理方案执行
    ├─ 历史数据：标记或隐藏违规内容
    └─ 新注册：立即生效拦截
```

#### 3.3.3 技术实现方案

**步骤1：更新敏感词级别**

```python
# 管理员升级敏感词级别
@router.post("/sensitive_word/upgrade_level")
async def upgrade_sensitive_word_level(
    word_id: int,
    new_level: int = 2,  # 升级为禁止级别
    db: Session = Depends(get_db)
):
    # 1. 更新敏感词级别
    word = db.query(SensitiveWord).filter(SensitiveWord.id == word_id).first()
    old_level = word.level
    word.level = new_level
    db.commit()
    
    # 2. 触发全量扫描任务（异步）
    await scan_all_users_for_sensitive_word(word.word, old_level, new_level)
    
    return {"msg": "敏感词级别已升级，正在扫描受影响的数据"}
```

**步骤2：全量用户扫描**

```python
async def scan_all_users_for_sensitive_word(word: str, old_level: int, new_level: int):
    """扫描所有用户，处理包含敏感词的用户"""
    db = next(get_db())
    
    # 查找昵称包含敏感词的用户
    nickname_users = db.query(User).filter(User.nickname.contains(word)).all()
    
    # 查找登录名包含敏感词的用户
    login_name_users = db.query(User).filter(User.login_name.contains(word)).all()
    
    # 合并用户列表（去重）
    affected_users = list(set(nickname_users + login_name_users))
    
    # 记录扫描结果
    scan_result = {
        "word": word,
        "old_level": old_level,
        "new_level": new_level,
        "affected_user_count": len(affected_users),
        "affected_users": [
            {"id": u.id, "login_name": u.login_name, "nickname": u.nickname}
            for u in affected_users
        ]
    }
    
    # 发送管理员通知
    await notify_admin(scan_result)
    
    # 根据新级别执行处理
    if new_level == 2:  # 升级为禁止级别
        for user in affected_users:
            await handle_forbidden_sensitive_word_user(user, word)
    
    db.close()
```

**步骤3：处理受影响用户**

```python
async def handle_forbidden_sensitive_word_user(user: User, word: str):
    """处理包含禁止级别敏感词的用户"""
    
    # 1. 强制重置昵称
    if word in user.nickname:
        new_nickname = generate_temp_nickname(user.id)
        user.nickname = new_nickname
        
        # 记录重置日志
        log = SensitiveWordLog(
            word=word,
            action="auto_reset_nickname",
            operator="system",
            details=f"用户ID:{user.id}, 原昵称重置为:{new_nickname}"
        )
        db.add(log)
    
    # 2. 强制重置登录名
    if word in user.login_name:
        new_login_name = generate_temp_login_name()
        user.login_name = new_login_name
        
        # 记录重置日志
        log = SensitiveWordLog(
            word=word,
            action="auto_reset_login_name",
            operator="system",
            details=f"用户ID:{user.id}, 原登录名重置为:{new_login_name}"
        )
        db.add(log)
    
    db.commit()
    
    # 3. 发送通知
    await notify_user_sensitive_word_reset(user, word)
```

**步骤4：历史数据处理**

```python
async def handle_historical_data(word: str):
    """处理包含敏感词的历史数据"""
    db = next(get_db())
    
    # 1. 扫描评论内容
    comments = db.query(Comment).filter(Comment.content.contains(word)).all()
    
    for comment in comments:
        # 标记评论为包含敏感词
        comment.has_sensitive_word = True
        comment.sensitive_word = word
        
        # 可选：隐藏评论或替换敏感词
        # comment.is_visible = False
        # comment.content = comment.content.replace(word, "***")
    
    db.commit()
    db.close()
```

## 六、总结

### 6.1 核心要点

1. **分级处理**：禁止级别立即处理，警告级别限期整改
2. **用户权益**：充分通知，给予整改期限，建立申诉机制
3. **系统自动化**：扫描、重置、通知自动化，减少人工干预
4. **渐进式执行**：从通知到限制再到强制重置，给用户缓冲时间

### 6.2 关键指标

| 指标 | 目标值 | 监控方式 |
|------|--------|----------|
| 禁止级别用户处理率 | 100% | 系统日志 |
| 警告级别用户整改率 | >80% | 数据库查询 |
| 用户投诉率 | <5% | 客服系统 |
| 申诉通过率 | >90% | 申诉系统 |

### 6.3 后续优化

1. **智能识别**：引入AI识别敏感词变体（如拼音、谐音）
