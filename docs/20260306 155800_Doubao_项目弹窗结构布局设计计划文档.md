# 一、弹窗整体框架布局

弹窗整体采用“蒙层+左右分栏半屏”布局（PC端），移动端适配全屏上下布局，核心原则为“视觉聚焦、分区清晰、风格统一”，具体布局规范如下：

## 1.1 蒙层布局

- 覆盖范围：全屏覆盖浏览器视口（PC端）/手机视口（移动端），无任何留白；

- 位置：位于弹窗下方，作为弹窗的背景兜底，不遮挡弹窗任何内容；

- 形态：无边界、无圆角，纯纯色背景，无渐变、无图案、无水印。

## 1.2 弹窗主体布局（PC端核心）

- 定位：固定定位，不随页面滚动变化，始终悬浮于视口上方、蒙层之上；

- 尺寸：高度=100vh（占满整个视口高度），宽度为视口宽度的75%（固定核心内容区宽度，避免屏幕尺寸变化导致布局错乱）；

- 分栏比例：左侧视觉区与右侧信息区宽度比为3:2，左侧聚焦核心视觉内容，右侧承载文字信息；

- 形态：仅左上角、左下角为12px圆角，右侧无圆角；无任何边框，与蒙层形成自然视觉区分；

- 背景：纯白色背景，与项目页面整体背景保持一致，确保视觉统一性。

## 1.3 移动端适配布局

- 布局切换：屏幕宽度＜992px时，放弃左右分栏，切换为“上下全屏布局”；

- 尺寸：宽度=100vw，高度=100vh，四个角均为12px圆角，无边框；

- 分区顺序：顶部为视觉区，下方为信息区，整体纵向排列，适配移动端浏览习惯。

# 二、弹窗内部分区布局（PC端）

弹窗内部按“顶部固定区→左右分栏区→底部固定区”的顺序布局，其中左右分栏区为核心内容区，顶部、底部固定区不随内容滚动变化，具体分区细节如下：

## 2.1 顶部固定区（全宽跨栏）

- 位置：弹窗最顶部，横跨左侧视觉区与右侧信息区，贯穿弹窗全宽；

- 尺寸：高度固定为60px，宽度=弹窗整体宽度，无左右边距、上下间距；

- 内容布局：仅包含1个关闭按钮，无其他冗余元素；

- 按钮位置：弹窗右上角，距离弹窗右侧20px、顶部20px，居中对齐；

- 区域特点：无背景色、无边框，不干扰下方内容展示，仅提供基础关闭功能的布局载体。

## 2.2 左侧核心视觉区（分栏左侧，占比60%）

- 位置：顶部固定区下方，占据弹窗左侧60%宽度，高度=100vh-130px（减去顶部60px+底部70px固定区高度）；

- 内容载体：用于展示项目核心视觉内容（如六爻排盘画像、高清图片等），无其他文字或功能元素；

- 内容适配：
        

    - 宽度：与左侧视觉区宽度100%一致，无左右内边距、外边距；

    - 高度：根据内容原始比例自适应，保证内容完整、清晰，不拉伸、不裁剪；

    - 兜底布局：内容加载前，显示浅灰色占位背景，占位背景尺寸与左侧视觉区完全一致。

- 区域特点：无任何边框、无额外装饰，最大化展示视觉内容，作为弹窗的视觉核心。

# 三、移动端弹窗内部分区布局

移动端弹窗为全屏上下布局，按“顶部关闭区→视觉区→信息区→底部互动区”的顺序纵向排列，所有区域均在全屏范围内，具体布局如下：

## 3.4 底部互动区

- 尺寸：高度固定为60px，宽度=100vw；

# 四、布局统一规范

## 4.1 边距规范

- PC端：弹窗内部各区域左右内边距统一为24px，元素间距统一为8px、16px两档；

- 移动端：弹窗内部各区域左右内边距统一为15px，元素间距统一为8px、12px两档；

- 所有区域无缝衔接，无额外间距，避免布局松散。

## 4.2 尺寸规范

- 固定区域高度统一：PC端顶部60px、用户信息区80px、互动区70px；移动端顶部50px、互动区60px；

- 可滚动区域高度自适应，根据内容和视口尺寸动态调整，不固定高度；

- 图标、按钮尺寸统一，避免出现尺寸混乱导致的布局失衡。

## 4.3 视觉统一规范

- 背景色统一：弹窗主体为纯白色，蒙层为纯半透明色，无其他杂色；

- 圆角统一：PC端仅弹窗左上角、左下角为12px圆角，移动端四个角均为12px圆角；

- 分隔线统一：所有分隔线均为1px浅灰色，无渐变、无阴影，仅用于区分不同区域。
