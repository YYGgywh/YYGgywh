# CSS 文件改造实施计划

## 文档信息

- **编制日期**：2026-03-07
- **适用范围**：YYG_paipan_Project 前端项目
- **改造目标**：将传统 CSS 文件改造为 CSS Modules 格式
- **预期周期**：2-3 周

---

## 一、改造任务概述

### 1.1 改造背景

当前项目存在严重的 CSS 污染问题，主要表现为：
- 类名冲突导致样式相互覆盖
- 全局样式难以维护和管理
- 组件样式耦合度高，影响开发效率

### 1.2 改造目标

1. **解决样式冲突**：通过 CSS Modules 实现样式隔离
2. **提高可维护性**：组件样式与组件紧密关联
3. **优化性能**：减少样式计算，提升渲染性能
4. **建立规范**：形成统一的 CSS Modules 开发规范
5. **支持主题开发**：通过 CSS 变量实现主题切换

### 1.3 改造原则

1. **渐进式改造**：分阶段进行，降低风险
2. **优先级排序**：先改造高频使用、影响范围大的组件
3. **保持兼容**：确保改造过程中功能不受影响
4. **充分测试**：每个文件改造后都需验证
5. **变量化设计**：所有样式属性值使用 CSS 变量表达
6. **避免重复定义**：优先使用全局变量，组件级变量仅定义特有属性
7. **层级分离**：桌面端与移动端样式完全分离，不在桌面端文件中写移动端适配

**CSS 变量实施方法**：

#### 变量层级架构

```
全局变量（styles/variables.css）
├── 颜色系统：--primary-color, --text-primary, --bg-white
├── 间距系统：--spacing-xs, --spacing-sm, --spacing-md
├── 字体系统：--font-size-base, --line-height-base
└── 动画系统：--transition-fast, --transition-normal

页面级变量（pages/*.module.css）
└── :root { --home-primary-color: #1890ff; }

组件级变量（components/**/*.module.css）
└── .componentRoot { --card-bg-color: #ffffff; }
```

#### 实施标准

```css
/* 1. 优先使用全局变量，提供 fallback 值 */
.waterfallCard {
  background: var(--bg-white, #ffffff);
  padding: var(--spacing-md, 16px);
  border-radius: var(--radius-lg, 8px);
  transition: all var(--transition-fast, 0.2s) ease;
}

/* 2. 仅在需要覆盖时才定义组件级变量 */
.waterfallCard {
  /* 组件特有变量 */
  --card-custom-shadow: 0 4px 12px rgba(0, 0, 0, 0.15);
  
  /* 使用变量 */
  box-shadow: var(--card-custom-shadow);
}

/* 3. 页面级变量使用前缀避免冲突 */
/* HomePage.desktop.module.css */
:root {
  --home-container-max-width: 1200px;
  --home-tag-gap: 12px;
}

.homeContainer {
  max-width: var(--home-container-max-width);
}
```

#### 变量命名规范

| 层级 | 命名格式 | 示例 | 说明 |
|------|---------|------|------|
| 全局 | `--category-property` | `--primary-color` | 全局通用 |
| 页面 | `--page-property` | `--home-tag-gap` | 页面特有 |
| 组件 | `--component-property` | `--card-bg-color` | 组件特有 |

**优先级**：⭐⭐⭐⭐⭐（最高）

**理由**：
- 首页核心组件，用户访问频率最高
- 存在类名冲突风险（`.card-title`、`.user-info` 等）
- 改造后可显著提升首页渲染性能
- 作为 CSS 变量实施的示范案例

**预计工作量**：2 小时

## 三、各文件改造的优先级说明

### 优先级定义

| 优先级 | 标识 | 说明 | 处理策略 |
|--------|------|------|----------|
| 最高 | ⭐⭐⭐⭐⭐ | 核心组件，影响范围广 | 立即改造 |
| 高 | ⭐⭐⭐⭐☆ | 重要组件，用户体验关键 | 优先改造 |
| 中 | ⭐⭐⭐☆☆ | 通用组件，复用率高 | 计划改造 |
| 低 | ⭐⭐☆☆☆ | 普通组件，影响范围小 | 后期改造 |

### 优先级评估标准

1. **影响范围**：组件在项目中使用的页面数量
2. **样式问题**：类名冲突风险、样式复杂度

## 六、改造注意事项

### 5.1 命名规范

- **桌面端文件**：`ComponentName.desktop.module.css`
- **移动端文件**：`ComponentName.mobile.module.css`
- **类名规范**：使用小驼峰命名（如 `cardTitle`、`userInfo`）
- **CSS 变量命名**：使用 `--component-name-property` 格式（如 `--card-bg-color`）

### 5.2 样式导入

```javascript
// 改造前
import './Component.css';

// 改造后
import styles from './Component.desktop.module.css';
```

### 5.3 类名引用

```javascript
// 改造前
<div className="card-title">

// 改造后
<div className={styles.cardTitle}>
```

### 5.4 动态类名

```javascript
// 改造前
<div className={`card-title ${isActive ? 'active' : ''}`}>

// 改造后
<div className={`${styles.cardTitle} ${isActive ? styles.active : ''}`}>
```

### 5.5 CSS 变量使用规范

```css
/* 变量定义位置 */
.componentRoot {
  /* 颜色变量 */
  --component-bg-color: #ffffff;
  /* 尺寸变量 */
  --component-padding: 16px;
  /* 动画变量 */
  --component-transition: 0.3s;
}

/* 变量使用 */
.componentRoot {
  background: var(--component-bg-color);
  padding: var(--component-padding);
  transition: all var(--component-transition);
}
```

### 5.6 变量层级管理

```css
/* 全局变量 - styles/variables.css */
:root {
  --primary-color: #1890ff;
  --spacing-md: 16px;
}

/* 页面级变量 - pages/HomePage.desktop.module.css */
:root {
  --home-primary-color: #1890ff;  /* 页面特有 */
  --home-container-max-width: 1200px;
}

/* 组件级变量 - components/Card.module.css */
.cardRoot {
  --card-custom-shadow: 0 4px 12px rgba(0, 0, 0, 0.15);
}
```

### 5.7 样式精简原则

**根组件（App）精简标准**：
- ✅ 保留：核心布局框架（Grid、Flex 等）
- ❌ 删除：未使用的占位符样式
- ❌ 删除：@media 响应式查询（移动端样式分离到 .mobile.module.css）
- ❌ 删除：重复的 CSS 变量定义

**页面组件精简标准**：
- ✅ 保留：页面特有变量（使用 `--page-*` 前缀）
- ✅ 保留：页面特有布局样式
- ❌ 删除：与全局变量重复的定义
- ❌ 删除：未使用的样式类

### 5.8 主题开发支持

```css
/* 通过覆盖全局变量实现主题切换 */
.dark-theme {
  --primary-color: #1890ff;
  --bg-white: #1f1f1f;
  --text-primary: #ffffff;
  --shadow-sm: 0 1px 3px rgba(255, 255, 255, 0.1);
}

/* 组件自动适应主题 */
.card {
  background: var(--bg-white);
  color: var(--text-primary);
  box-shadow: var(--shadow-sm);
}
```
