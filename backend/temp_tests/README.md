# 临时测试目录说明

## 目录用途
- 🔬 **实验性代码测试** - 测试新功能或算法
- 🧪 **功能验证** - 验证特定功能的正确性
- 📝 **代码片段测试** - 测试小的代码片段
- 🔍 **调试辅助** - 用于调试和问题排查

## 使用规则
1. **临时性质** - 此目录下的文件可能随时被删除
2. **独立运行** - 每个测试文件应该可以独立运行
3. **清晰命名** - 文件名应明确说明测试内容
4. **及时清理** - 测试完成后及时清理不需要的文件

## 文件命名规范
- `test_功能名_日期.py` - 例如：`test_lunar_conversion_20241215.py`
- `experiment_实验名.py` - 例如：`experiment_ganzhi_calculation.py`
- `debug_问题描述.py` - 例如：`debug_jieqi_accuracy.py`

## 示例文件
- `test_quick_function.py` - 快速功能测试
- `experiment_new_algorithm.py` - 新算法实验
- `debug_specific_issue.py` - 特定问题调试
