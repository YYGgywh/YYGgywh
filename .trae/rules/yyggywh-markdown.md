
# Markdown 文档命名规则

## 描述
本规则用于规范项目中生成的 Markdown 文档命名方式，确保文档命名具有清晰的时间标识和可读性，便于文档管理与追溯。

## 命名格式要求
生成的 Markdown 文档名必须遵循以下格式：`YYYYMMDD HHMMSS_文档名.md`，其中：
 - 生成文档前先刷新时间戳。
 - `YYYYMMDD HHMMSS` 代表生成该文档时的年、月、日 时、分、秒。
 - `_` 为分隔符，用于区分日期时间与文档名。
 - 年月日与时分秒之间以空格连接。
 - 具体时间以生成该文档时的系统实际时间（时间戳）为准。
 - `文档名` 为对该文档内容的简要概括，尽量简洁明了且不包含特殊字符（除了下划线、连字符等常见分隔符）。
 - 无需在文件头部添加注释，命名时直接包含相关信息即可。
例如，若在 2024 年 10 月 15 日 14 时 30 分 05 秒生成一份关于项目进度汇报的文档，其命名应为：`20241015 143005_项目进度汇报.md`

alwaysApply: false
globs: ["*.md"]  // 仅对 Markdown 文件生效
