# 文件头部注释规范

## 描述
项目中所有文件都应在头部添加注释，说明文件相关信息，确保代码可维护性和可读性，便于团队成员理解文件用途、作者、创建及修改时间等内容。

## 注释格式要求
所有文件头部需使用多行注释形式，内容包括文件路径、功能描述、作者、创建时间、最后修改时间等信息。示例格式如下（注意不同文件格式对注释格式的要求）。：
```javascript
/*
 * @file            文件相对项目根目录的路径
 * @description     文件功能描述
 * @author          作者姓名及联系方式
 * @createTime      创建时间，格式为YYYY-MM-DD HH:mm:ss
 * @lastModified    最后修改时间，格式为YYYY-MM-DD HH:mm:ss
 * Copyright © All rights reserved
*/

# Markdown 文档命名规则

## 描述
本规则用于规范项目中生成的 Markdown 文档命名方式，确保文档命名具有清晰的时间标识和可读性，便于文档管理与追溯。

## 命名格式要求
生成的 Markdown 文档名必须遵循以下格式：`YYYYMMDD HHMMSS_文档名.md`，其中：
 - `YYYYMMDD HHMMSS` 代表生成该文档时的年、月、日 时、分、秒。
 - `_` 为分隔符，用于区分日期时间与文档名。
 - 年月日与时分秒之间以空格连接。
 - 具体时间以生成该文档时的系统实际时间（时间戳）为准。
 - `文档名` 为对该文档内容的简要概括，尽量简洁明了且不包含特殊字符（除了下划线、连字符等常见分隔符）。
 - 无需在文件头部添加注释，命名时直接包含相关信息即可。
例如，若在 2024 年 10 月 15 日 14 时 30 分 05 秒生成一份关于项目进度汇报的文档，其命名应为：`20241015 143005_项目进度汇报.md`
